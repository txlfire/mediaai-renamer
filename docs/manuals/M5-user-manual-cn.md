# MediaAI Renamer M5 用户手册

版本：`0.5.3`
阶段：M5 NAS / SMB / NFS 共享目录支持

## 1. 功能概览

M5 支持在媒体源中选择路径类型：

- 本地路径：服务所在机器上的普通目录。
- Windows UNC / SMB：形如 `\\server\share\media` 的共享路径。
- 已挂载路径 / NFS：已经由操作系统、NAS 或 Docker 映射好的目录。

系统只访问服务端能看到的路径。浏览器所在电脑能访问某个目录，不代表后端服务也能访问。

## 2. 添加媒体源

进入“媒体源”页面，填写名称、路径类型和目标路径。

本地路径：

- 可点击“选择目录”浏览服务所在 Windows 主机的目录。
- 也可手动输入路径。

Windows UNC / SMB：

- 输入 `\\server\share\folder` 格式路径。
- 可填写 SMB 账号和密码。
- 密码加密保存，不在列表和日志中明文显示。
- 选择该类型时不显示“选择目录”按钮。

已挂载路径 / NFS：

- 输入服务端或容器内可访问的挂载路径，例如 `/mnt/media`。
- 可填写 NFS 服务器和导出目录用于说明。
- 不提供用户名和密码输入框。
- 系统不会自动挂载 NFS。

## 3. 测试连接

媒体源列表和编辑弹窗均提供“测试连接”。

测试内容包括：

- 路径是否存在。
- 是否为目录。
- 当前服务进程是否可读、可写。
- SMB / UNC 是否可访问。
- NFS / 已挂载路径是否可访问，并尽量识别文件系统类型。

测试失败不会删除已保存配置。请根据提示检查路径、网络、挂载状态和服务运行用户权限。

## 4. 扫描与重命名

发起扫描前，系统会先检查媒体源启用状态和路径可访问性。

扫描过程中：

- 支持分批扫描、批间隔、递归扫描、跳过隐藏文件和最小文件阈值。
- 低于最小阈值的文件进入“待处理文件”列表。
- 部分文件跳过或失败时，任务显示“部分完成”。
- 共享目录中断时，任务显示“连接中断”或失败原因。

重命名前：

- 必须先执行 dry-run。
- 系统检查源文件、目标文件、目标目录写权限和同名冲突。
- 不覆盖已有文件。

## 5. Docker / NAS 注意事项

Docker 中使用共享目录时，容器内路径必须能访问文件。例如：

```yaml
volumes:
  - /mnt/nfs/media:/app/media
```

此时媒体源路径应填写容器内路径：

```text
/app/media
```

而不是宿主机路径 `/mnt/nfs/media`。

## 6. 常见问题

路径不存在：

- 检查路径是否写给后端服务所在机器，而不是浏览器所在机器。
- Docker 场景检查 volume 映射是否正确。

权限不足：

- 检查运行后端服务的用户是否有读写权限。
- NAS / NFS 场景检查 UID/GID 映射。

SMB 连接失败：

- 检查 UNC 格式是否正确。
- 检查 Windows 主机是否已经能访问该共享。
- 检查账号密码和共享权限。

NFS 连接失败：

- 先在系统层完成挂载。
- 检查 `/etc/fstab`、网络、防火墙和 NAS 导出权限。

## 7. 后续候选协议

WebDAV、FTP、SFTP、S3 暂不支持真实扫描和重命名。后续如果接入，需要单独设计远程目录枚举、凭据管理、远程重命名语义、失败恢复和审计策略。

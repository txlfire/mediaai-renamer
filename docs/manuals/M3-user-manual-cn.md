# MediaAI Renamer M3 用户手册

版本：v0.3.0 正式版
阶段：M3 安全重命名
日期：2026-06-26

## 1. 当前可以做什么

M3 支持在命名预览结果确认后，对真实文件执行安全重命名。

系统会先进行冲突检测，只有检测通过的文件才允许执行。重命名只会修改原目录内的文件名，不会移动文件夹，也不会覆盖已有文件。

## 2. 使用前准备

在执行重命名前，需要先完成：

1. 在“媒体源”中添加目标目录。
2. 在“扫描任务”中完成扫描。
3. 在“命名预览”中生成预览结果。
4. 检查或编辑目标文件名。

## 3. 进行冲突检测

进入“命名预览”页面后：

1. 选择媒体源和扫描任务。
2. 点击“查询”加载该任务的命名预览。
3. 勾选需要处理的预览记录。
4. 点击“冲突检测”。

系统会打开结果弹窗，展示本次检测的总数、可执行数量、冲突数量、已重命名数量和失败数量。

## 4. 查看冲突原因

常见冲突包括：

- 源文件不存在。
- 目标文件名为空。
- 目标文件名包含目录分隔符。
- 目标文件已存在。
- 同一批次中多个文件使用了相同目标路径。
- 预览记录状态不允许重命名。

有冲突的条目不会被执行，需要先回到命名预览列表修正目标文件名，或重新扫描确认文件状态。

## 5. 确认重命名

当检测结果中存在可执行项时，可以点击“确认重命名”。

执行后：

- 成功项状态变为“已重命名”。
- 媒体文件路径和文件名会同步更新。
- 失败项会保留失败原因。
- 命名预览列表会刷新。

## 6. 注意事项

- M3 不支持撤销，请在执行前确认目标文件名。
- M3 不会覆盖已有文件。
- M3 不会移动文件到其他目录。
- 如果文件被其他程序占用，可能会执行失败。
- 如果执行前真实文件已被移动或删除，对应条目会失败。

## 7. 版本限制

当前版本暂不支持：

- 批量移动目录。
- 回滚重命名。
- 权限审批。
- AI 自动识别元数据。
- 重命名历史页面。

这些能力会进入后续阶段规划。

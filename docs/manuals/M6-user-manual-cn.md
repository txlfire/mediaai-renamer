# MediaAI Renamer M6 用户手册

版本：`0.6.8`
覆盖范围：M6 AI 智能解析与外部提交保护
更新日期：2026-07-03

## 1. M6 新增能力

M6 新增以下可用能力：

- AI 智能解析配置。
- DeepSeek / OpenAI-compatible 连接测试。
- 敏感词设置。
- 外部提交拦截记录。
- 重命名预览单条 AI 解析。
- AI 候选确认回填命名预览。
- 待处理文件单条 AI 解析候选展示。

AI 不会自动扫描全库，也不会直接重命名真实文件。

## 2. 配置 AI 智能解析

进入“系统设置”，打开“AI 智能解析”分类。

可配置：

- 启用 AI 智能解析。
- 服务商，当前默认 DeepSeek。
- 模型名，例如 `deepseek-chat`。
- API Key。
- Base URL。
- 接口超时时间。
- 最大重试次数。

保存后点击“测试连接”验证配置。测试状态栏会显示成功或失败，并显示完整更新时间。

## 3. 配置敏感词

进入“系统设置”，打开“敏感词设置”。

可配置：

- 是否启用默认敏感词库。
- 默认影视风险词。
- 自定义敏感词。

多个敏感词使用 `||` 分隔，例如：

```text
色情||暴力||AV
```

命中文件名、路径或解析标题后，系统不会向 TMDB、IMDb、AI 或后续外部刮削源提交该条目。

## 4. 查看拦截记录

敏感词设置页提供外部提交拦截记录列表。

列表展示：

- 文件名。
- 目标服务。
- 命中摘要。
- 拦截状态。
- 拦截时间。
- 用户处理记录。

用户可以将记录标记为已改名、忽略、确认风险继续提交或归档。确认风险继续提交只对当前记录和当前目标服务生效，不会关闭全局敏感词规则。

## 5. 在重命名预览中使用 AI

进入“重命名”页面，选择扫描任务并生成预览。

对单条记录使用 AI：

1. 点击行操作中的“AI 解析”按钮。
2. 等待 AI 返回候选。
3. 在弹窗中查看标题、类型、年份、季集、置信度和原因。
4. 点击“选择回填”。
5. 表格中的目标文件名更新。

回填后不会立即修改真实文件。仍需点击“重命名”或“全部重命名”，并通过 dry-run 检查后才会执行真实文件改名。

## 6. 在待处理文件中使用 AI

待处理文件列表支持单条 AI 解析。

使用方式：

1. 打开“重命名”页面。
2. 切换到“待处理文件列表”。
3. 点击行操作中的“AI 解析”按钮。
4. 查看 AI 候选结果。

当前阶段待处理文件没有直接回填按钮。若需要进入重命名流程，请先按实际情况调整文件或扫描配置，再重新扫描和生成命名预览。

## 7. 安全说明

AI 调用必须由用户主动点击触发。

以下情况不会调用 AI：

- AI 未启用。
- API Key 未配置。
- 文件名、路径或解析标题命中敏感词。
- 后端无法通过结构化校验。

AI 输出只作为候选，系统会再次校验字段范围。非法输出不会自动回填。

## 8. 常见问题

### 8.1 AI 连接失败

检查：

- API Key 是否已保存。
- Base URL 是否正确。
- 模型名是否正确。
- 网络或代理是否可访问服务商接口。
- 超时时间是否过短。

### 8.2 AI 解析被拦截

进入“敏感词设置”查看拦截记录。你可以改名后重新匹配、忽略记录，或在明确风险后确认继续提交。

### 8.3 AI 回填后没有真实改名

这是正常行为。AI 只回填命名预览，不会直接修改真实文件。真实改名必须从重命名页面执行。

# MediaAI Renamer 用户手册

版本：`0.5.3`
覆盖范围：M1-M5 已完成能力
更新日期：2026-06-28

## 1. 产品用途

MediaAI Renamer 用于扫描本地或服务端可访问的影视目录，生成标准化命名预览，并在用户确认后安全重命名真实文件。

当前已支持：

- 媒体源管理。
- 本地目录选择。
- 分批扫描。
- 扫描任务和扫描结果查看。
- 命名预览生成、查询、编辑和排序。
- TMDB 元数据匹配。
- dry-run 冲突检测。
- 多选、全部和单条安全重命名。
- 待处理文件列表和批量迁移。
- 系统设置热更新。
- NAS / SMB / NFS 共享目录基础支持。
- 亮色、暗色和跟随系统主题。
- 操作日志查看。

当前主要限制：

- SMB / UNC 连接测试检查当前后端服务进程是否能访问共享路径；系统不会自动执行 `net use` 登录共享。
- NFS 需要用户先在操作系统、宿主机或 NAS 中完成挂载，本系统不会自动执行挂载命令。
- WebDAV、FTP、SFTP、S3 暂不支持真实扫描和重命名，仅作为后续候选协议。
- AI 智能解析将在 M6 开始开发。
- 高级命名规则、权限、回滚和审计属于后续阶段。

## 2. 启动和访问

本地开发推荐使用：

```powershell
npm run dev:lan
```

启动后访问：

```text
http://127.0.0.1:5173
```

后端默认端口为 `8970`，前端开发端口为 `5173`，Docker Web 端口为 `8971`。

## 3. 页面结构

页面左侧为导航栏，包含：

- 产品名称和 Logo。
- 媒体源。
- 扫描任务。
- 扫描结果。
- 命名预览。
- 系统设置。
- 后端状态。
- 语言选择占位。
- 版本信息和主题切换。
- 当前用户和退出入口。

页面右侧为操作台。点击左侧菜单后，右侧会切换到对应页面。

侧边栏支持展开和收缩：

- 展开时显示图标和文字。
- 收缩时只显示图标，鼠标悬浮可查看名称。

## 4. 主题和状态

主题按钮位于左下角版本信息附近，支持亮色和暗色切换，并可跟随系统主题。

后端状态使用图标和文字显示：

- 绿色：连接正常。
- 红色：连接异常。
- 灰色：未知或检测中。

语言选择当前默认显示中文，完整多语言切换后续实现。

## 5. 媒体源管理

进入“媒体源”页面后，可以管理需要扫描的目录。

### 5.1 新增媒体源

新增媒体源需要填写：

- 名称。
- 路径类型。
- 目标路径。
- 启用状态。

路径类型包括：

- 本地路径：服务所在机器可直接访问的目录。
- Windows UNC / SMB：形如 `\\NAS\Movies` 的共享路径。
- 已挂载路径 / NFS：Linux、NAS、Docker 或 NFS 已挂载到本地后的路径，如 `/mnt/media`。

当前保存时只校验必填项和基础格式，不强制校验路径有效性。路径是否可用可以通过“测试连接”入口确认。

### 5.2 本地目录选择

选择“本地路径”或“已挂载路径 / NFS”时，可以点击“选择目录”，在弹窗中浏览服务所在机器可访问的目录。

目录选择弹窗会展示：

- 当前路径。
- 上一级目录。
- 子目录列表。
- 可读、可写权限状态。

不可读目录会禁用进入。

### 5.3 SMB 路径

选择 Windows UNC / SMB 时：

- 目标路径需要手动输入。
- 路径必须以 `\\` 开头。
- 账号和密码只在 SMB 类型下展示。
- 密码加密保存，不在列表和接口响应中回显明文。

连接测试会检查当前服务进程是否能访问该 UNC 路径，并返回读写权限状态。系统不会自动执行 `net use` 登录共享；如果当前服务进程没有共享访问权限，需要先在系统层完成访问配置。

### 5.4 NFS / 已挂载路径

选择已挂载路径 / NFS 时：

- 不显示用户名和密码。
- 可填写 NFS 服务器和导出目录作为说明信息。
- 系统只访问本地挂载点，不自动执行 `mount`。
- Docker 场景需要先在宿主机完成挂载，再映射到容器内路径。
- Linux 环境下会尽量识别 `nfs` 或 `nfs4` 文件系统类型；无法确认时只提示，不在路径可访问时强制失败。

### 5.5 测试连接

新增和编辑媒体源时都可以点击“测试连接”。

测试内容包括：

- 路径是否存在。
- 是否为目录。
- 当前服务进程是否可读、可写。
- SMB / UNC 路径格式和当前进程访问能力。
- NFS / 已挂载路径访问能力，并尽量识别文件系统类型。

测试失败不会删除已保存配置，也不会阻止只保存必填项完整的媒体源。请根据提示检查路径、网络、挂载状态和服务运行用户权限。

### 5.6 编辑媒体源

点击媒体源名称或“修改”图标可打开编辑弹窗。

编辑时可以修改：

- 名称。
- 目标路径。
- SMB 账号和密码。
- NFS 服务器和导出目录。
- 启用状态。

路径类型保存后不能修改。如需变更路径类型，请新建媒体源。

如果修改目标路径，系统会提示该操作将清空该媒体源历史任务、扫描结果和命名预览数据。确认后会显示清理进度。

### 5.7 删除媒体源

支持单条删除和批量删除。删除媒体源会同时清理其关联的历史扫描任务、扫描结果和命名预览记录。

## 6. 扫描任务

进入“扫描任务”页面后：

1. 选择一个已启用媒体源。
2. 点击“开始扫描”。
3. 等待任务完成。
4. 在任务列表查看扫描统计。

任务列表包含：

- 任务 ID。
- 状态。
- 已扫描数量。
- 视频数量。
- 警告数量。
- 单批数量。
- 间隔。
- 开始时间。
- 结束时间。

扫描采用分批执行，默认每批 100 个文件，批间隔 1 秒。相关配置可在系统设置中调整。

任务行支持直接跳转：

- 查看扫描结果。
- 查看命名预览。

## 7. 扫描结果

进入“扫描结果”页面后，需要先选择媒体源和扫描任务，再点击“查询”。

扫描结果默认不自动加载，只有用户手动查询或从扫描任务跳转时才加载对应任务的数据。

列表字段包括：

- 序号。
- 文件名。
- 格式。
- 大小。
- 任务 ID。
- 路径。
- 修改时间。

支持分页、排序、详情弹窗和重置。重置会清空查询条件和当前列表数据。

## 8. 命名预览

进入“命名预览”页面后，可以对扫描结果生成标准目标文件名。

页面包含两个标签页：

- 命名预览列表。
- 待处理文件列表。

### 8.1 生成预览

1. 选择媒体源。
2. 选择扫描任务。
3. 点击“生成预览”。
4. 点击“查询”查看预览结果。

命名预览默认不会在进入页面时自动加载；只有查询、生成预览或从扫描任务跳转时才加载。

### 8.2 预览状态

常见状态包括：

- 可使用：系统生成了可执行的目标文件名。
- 需检查：解析结果不完整，需要人工确认。
- 已编辑：用户手动修改过目标文件名。
- 已重命名：真实文件已完成重命名。

### 8.3 编辑目标文件名

点击操作列中的“编辑”图标，可以修改目标文件名。

规则：

- 只能编辑文件名，不能输入目录路径。
- 如果没有输入扩展名，系统会自动补回原扩展名。
- 手动编辑会保存到预览记录中，并优先用于最终重命名。

### 8.4 TMDB 匹配

点击操作列中的 TMDB 匹配图标可对单条预览执行元数据匹配。

匹配结果包括：

- 高置信：自动回填目标文件名。
- 需确认：打开候选弹窗，由用户选择候选后回填。
- 未匹配：保留本地解析结果。

手动编辑的目标文件名仍优先用于最终重命名。

## 9. 安全重命名

真实重命名必须从命名预览页面发起。

支持三种方式：

- 勾选多条记录后点击“执行重命名”。
- 点击“全部重命名”。
- 点击单行操作列中的“执行重命名”图标。

执行前系统会先检查目标文件名是否为空。如果存在空目标文件名，可以返回编辑，也可以一键剔除空名条目后继续。

空名检查通过后，系统会执行 dry-run 冲突检测。常见冲突包括：

- 源文件不存在。
- 目标文件名为空。
- 目标文件名包含目录分隔符。
- 目标文件已存在。
- 同一批次中多个文件使用相同目标路径。
- 预览状态不允许重命名。

只有 dry-run 通过的条目才会进入真实重命名。系统不会覆盖已有文件，也不会移动目录。

## 10. TMDB 设置

进入“系统设置”，打开“TMDB 元数据刮削”分类。

可配置：

- 启用 TMDB。
- TMDB V4 只读访问令牌。
- TMDB V3 API Key。
- 请求语言。
- 地区。
- 接口超时时间。
- 站点优先级。
- 最小扫描文件阈值。

系统优先使用 V4 令牌；V4 失败后自动降级使用 V3 API Key。

保存后可点击“测试连接”验证配置。成功时显示“连接成功！信息有效！”，失败时显示对应错误和中文提示。

## 11. 待处理文件

待处理文件列表位于命名预览页面的标签页中，主要展示低于最小扫描文件阈值或无需刮削的文件。

支持：

- 单条移除任务记录。
- 清空列表。
- 批量迁移真实文件。

移除和清空只删除待处理任务记录，不删除源文件。批量迁移会移动真实文件，请确认目标目录正确。

## 12. 系统设置

系统设置用于管理可热更新的系统参数。

分类包括：

- TMDB 元数据刮削。
- 文件扫描配置。
- 解析重命名配置。
- 系统运维配置。
- 共享目录配置。

保存后无需重启服务。部分配置只影响后续任务，不影响已经运行中的任务。

## 13. 操作日志

在扫描任务页面可打开操作日志。日志用于辅助排查扫描、预览、重命名和系统运行问题。

敏感信息规则：

- 不记录明文 API Key。
- 不记录明文密码。
- 不记录完整 token。
- SMB 凭据必须脱敏。

## 14. 列表通用规则

系统内列表统一遵循：

- 默认 10 条/页。
- 支持 50 条/页。
- 支持展示全部。
- 文本靠左。
- 日期、数字和状态类字段按页面规则居中。
- 文件大小展示为自然单位。
- 时间不展示毫秒和时区。
- 长文本省略后可悬浮查看完整内容。
- 无数据时显示“暂无匹配数据”。

除媒体源页面外，扫描任务、扫描结果和命名预览页面默认不做初始数据加载。已有查询数据时，切换菜单返回页面不会主动清空列表。

## 15. 常见问题

### 15.1 后端连接异常

检查：

- 后端服务是否启动。
- 后端端口是否为 `8970`。
- 页面是否可以访问 `/api/health`。

### 15.2 保存媒体源失败

检查：

- 名称是否为空。
- 目标路径是否为空。
- UNC 路径是否以 `\\` 开头。
- SMB 类型是否填写账号。
- NFS 类型是否填写服务器和导出目录。
- 路径是否与已有媒体源重复。

### 15.3 已停用媒体源无法扫描

这是正常限制。请先在媒体源页面启用该媒体源，再发起扫描。

### 15.4 扫描后没有结果

检查：

- 目录中是否有支持的视频格式。
- 最小扫描文件阈值是否过高。
- 文件是否进入了待处理列表。
- 扫描任务是否已完成。

### 15.5 TMDB 匹配失败

检查：

- TMDB 是否已启用。
- V4 令牌或 V3 API Key 是否正确。
- 网络是否可以访问 TMDB。
- 代理是否配置正确。
- 接口超时时间是否过短。

### 15.6 重命名失败

检查：

- 源文件是否仍存在。
- 目标文件名是否为空。
- 目标文件是否已存在。
- 文件是否被播放器、下载器或其他程序占用。
- 当前服务进程是否有写权限。

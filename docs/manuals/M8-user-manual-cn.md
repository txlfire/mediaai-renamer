# MediaAI Renamer M8 用户手册

版本：`0.8.0`
覆盖范围：M8 命名规则高级配置二期
更新日期：2026-07-08

## 1. 功能范围

M8 用于增强系统设置中的命名规则配置能力。用户可以测试模板、查看差异、导入导出配置，并在命名预览中识别历史记录是否由旧模板生成。

当前支持：

- 电影和剧集命名模板分别配置。
- 模板版本号和更新时间展示。
- 模板导出和导入校验。
- 规则测试，返回生成结果、字段命中和格式告警。
- 保存前差异预览。
- 中文标题、英文标题、原始标题等双语字段。
- 命名预览列表展示旧模板生成和版本未知提示。

## 2. 进入入口

进入“系统设置”，打开“解析重命名配置”分类，在命名规则区域配置电影模板和剧集模板。

修改模板后，建议先执行：

1. 规则测试。
2. 差异预览。
3. 保存配置。
4. 回到命名预览页重新生成预览。

## 3. 规则测试

规则测试用于验证当前编辑中的模板，不会写入正式命名预览表。

测试结果包含：

- 生成结果：按当前模板生成的目标文件名。
- 字段命中：模板字段是否在样例数据中有值。
- 格式告警：缺少字段、非法字符清理或空结果等提示。
- 模板版本：当前生效模板版本。

## 4. 差异预览

差异预览用于对比当前生效模板和待保存模板的输出差异。

差异预览不会修改历史预览记录，也不会修改真实文件。

## 5. 模板导入导出

导出会包含电影模板、剧集模板、版本号、更新时间和结构版本。

导入时会校验 JSON 结构、字段定义和格式参数。非法模板不会覆盖当前生效配置，用户需要修正后重新导入。

## 6. 历史预览提示

命名预览列表会根据模板版本显示状态：

- 当前模板生成：不额外提示。
- 旧模板生成：目标文件名旁显示“旧模板生成”。
- 版本未知：旧库或旧记录未保存模板版本时显示“版本未知”。

这些提示只用于判断是否需要重新生成预览，不会阻断编辑、TMDB 匹配、AI 匹配、dry-run 或安全重命名。

## 7. 注意事项

- 模板保存后不会自动批量改写历史预览。
- 手动编辑过的目标文件名仍受重新生成确认保护。
- AI、TMDB 和人工候选回填目标名时，会保留或刷新模板版本状态。
- 真实文件改名仍必须经过 dry-run 和用户确认。

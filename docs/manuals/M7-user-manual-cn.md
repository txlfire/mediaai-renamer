# MediaAI Renamer M7 用户手册

版本：`0.7.9`
覆盖范围：M7 增量扫描与标题识别增强
更新日期：2026-07-07

## 1. M7 新增能力

M7 交付以下能力：

- 全量 / 增量扫描模式。
- `scan_file_index` 历史索引。
- 扫描模式建议。
- 新增 / 修改 / 跳过 / 缺失统计。
- 扫描失败回滚保护，失败不污染历史索引。
- 标题识别模式配置。
- 上级文件夹标题兜底、冲突提示和详情展示。
- TMDB / AI 匹配来源切换。
- TMDB→AI 后备匹配。
- 手动排除到待处理文件列表。

## 2. 使用增量扫描

进入“扫描任务”页面后：

1. 选择媒体源。
2. 查看系统给出的扫描模式建议。
3. 首次扫描选择“全量扫描”。
4. 已存在历史索引时可选择“增量扫描”。
5. 扫描完成后查看新增、修改、跳过、缺失和已建索引数量。

增量扫描只会把新增或变化文件写入本次扫描结果。未变化文件只更新索引状态，不重复进入本次结果列表。

## 3. 配置标题识别模式

进入“系统设置 -> 解析重命名配置”，可选择以下模式：

- `file_name_first`：优先使用文件名标题。
- `parent_folder_fallback`：文件名标题不足时，使用上级文件夹标题补位。
- `parent_folder_first`：优先使用上级文件夹标题。
- `manual_only`：只展示候选，不自动替换。

当文件名标题与上级文件夹标题不一致时，系统会保留文件名标题，并把预览标记为“需检查”。

## 4. 在重命名页使用新匹配链路

重命名页顶部支持两类选择：

- 匹配模式：仅 TMDB、仅 AI、TMDB→AI。
- 匹配来源：解析标题、原文件名、上级文件夹标题。

适用说明：

- 标题不完整时，优先尝试“上级文件夹标题”。
- 只想看 AI 候选时选择“仅 AI”。
- 先走 TMDB、未匹配或低置信再走 AI 时选择“TMDB→AI”。

批量 AI 和 TMDB→AI 执行前会显示当前 Provider、模型、Base URL、处理数量、提交范围和 Token 成本提示。

## 5. 查看冲突和候选标题

命名预览详情中新增展示：

- 标题来源。
- 上级文件夹标题。
- 当前识别模式。
- 标题冲突原因。

这几项用于判断是否需要人工编辑目标文件名，或切换匹配来源重新执行 TMDB / AI 匹配。

## 6. 手动排除与待处理

预览列表支持：

- 单条排除。
- 选中批量排除。

排除后文件不会被删除，只会进入“待处理文件列表”，并记录原因为“手动排除”。

## 7. 常见问题

### 7.1 增量扫描后为什么本次结果很少

这是正常情况。未变化文件会计入“跳过”，不会重复写入本次扫描结果。

### 7.2 扫描失败后历史索引会不会丢

不会。M7 已补充失败回滚保护，失败任务只更新任务状态和错误信息，不会覆盖已有历史索引。

### 7.3 上级文件夹标题为什么没有自动替换

常见原因：

- 当前模式是 `file_name_first` 或 `manual_only`。
- 文件名标题与上级文件夹标题冲突，系统已标记为“需检查”。
- 当前文件就位于媒体源根目录下，没有可用的上级文件夹标题候选。

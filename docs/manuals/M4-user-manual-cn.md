# MediaAI Renamer M4 用户手册

版本：v0.4.0 预览版
阶段：M4 TMDB 元数据刮削与文件运维
日期：2026-06-26

## 1. 当前可以做什么

M4 增加了 TMDB 元数据匹配和待处理文件列表。

你可以在系统设置中配置 TMDB，随后在命名预览页面对单条预览执行 TMDB 匹配。高置信结果会自动回填目标文件名，低置信结果会打开候选弹窗，由你手动选择。

M4 还支持按文件大小阈值过滤扫描结果。小于阈值的视频不会进入正常重命名队列，而是进入待处理文件列表。

## 2. 配置 TMDB

进入“系统设置”页面：

1. 打开“TMDB 元数据刮削”分类。
2. 启用 TMDB。
3. 填写 TMDB API Key。
4. 按需设置请求语言、地区和超时时间。
5. 点击保存。

保存后无需重启服务。密钥保存后会脱敏显示。

如果未启用 TMDB 或未配置 API Key，系统会继续使用本地解析，不影响扫描、预览和重命名。

## 3. 配置文件最小扫描阈值

在“系统设置”中可以设置“最小扫描文件阈值（Byte）”。

- 设置为 `0`：所有识别到的视频文件都进入扫描结果。
- 设置为大于 `0`：小于该大小的视频文件会进入待处理列表。

该配置热生效，下一次扫描时使用新值。

## 4. 使用 TMDB 匹配

进入“命名预览”页面：

1. 选择媒体源和扫描任务。
2. 点击“查询”加载预览结果。
3. 在某一行的操作列点击“TMDB 匹配”图标。

匹配结果分为三类：

- 高置信：自动回填目标文件名。
- 需确认：打开候选结果弹窗，选择合适候选后回填。
- 未匹配：保留本地解析结果，不影响后续手动编辑或重命名。

## 5. 查看匹配信息

命名预览列表会展示：

- 元数据匹配状态。
- 匹配度百分比。
- 详情弹窗中的来源、分数和状态。

可以继续编辑目标文件名。手动编辑仍然优先用于最终重命名。

## 6. 待处理文件列表

待处理文件列表位于命名预览页面下方。

当前会展示因尺寸过滤进入待处理队列的文件，包括：

- 标签。
- 文件名。
- 源路径。
- 文件大小。
- 扫描时间。

## 7. 处理待处理文件

支持三种操作：

1. 单条移除：只移除待处理任务记录，不删除源文件。
2. 清空列表：清空当前筛选下的待处理任务记录，不删除源文件。
3. 批量迁移：勾选文件，输入目标文件夹路径，将真实文件移动到目标目录。

迁移前请确认目标文件夹存在，且目标目录中没有同名文件。

## 8. 注意事项

- TMDB 是增强能力，关闭后不影响本地解析。
- 网络异常、TMDB 超时或密钥缺失时，系统会降级，不阻断任务。
- 待处理文件的移除和清空不会删除真实文件。
- 批量迁移会移动真实文件，请确认目标路径。
- M4 暂不提供迁移撤销功能。

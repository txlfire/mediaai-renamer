# MediaAI Renamer M9 用户手册

版本：`0.9.1`
覆盖范围：M9 权限、历史、回滚、审计和任务治理
更新日期：2026-07-09

## 1. 功能范围

M9 补齐治理能力，让高风险操作具备登录身份、权限限制、审计记录、过程日志和回滚路径。

当前支持：

- 本地管理员初始化、登录、退出和当前用户查询。
- 用户信息维护、启停用户、重置临时密码和直接权限勾选。
- 系统设置、媒体源写操作、扫描创建、真实重命名、回滚、外部提交等敏感操作权限守卫。
- 登录、用户维护、配置保存、媒体源变更、外部提交覆盖、真实重命名和回滚审计。
- 扫描、重命名 dry-run、真实重命名和回滚的操作运行日志。
- 基于成功重命名批次创建回滚计划，先 dry-run 后执行。
- 任务治理页面统一查看扫描任务、重命名批次和回滚计划，支持归档已处理任务并按需恢复。

## 2. 登录和管理员初始化

系统首次启用认证后，如果用户表为空，可以创建首个本地管理员。

初始化后：

- 管理员需要登录后才能执行敏感写操作。
- 登录态过期后，页面会提示重新登录。
- 默认管理员密码登录后会持续提示修改密码。
- 密码只保存哈希，不保存明文。

## 3. 用户和权限

进入“系统设置”的用户信息维护区域，可以维护本地用户。

可操作内容：

- 新增用户。
- 修改显示名称。
- 启用或停用用户。
- 重置临时密码。
- 勾选直接权限。

首个管理员默认拥有全部权限。后续用户按权限列表控制，不引入角色模型。

常见权限包括：

- `settings:write`：保存系统设置和维护用户。
- `source:write`：新增、编辑、删除媒体源。
- `scan:run`：创建扫描任务。
- `rename:execute`：执行真实重命名。
- `rollback:execute`：创建、dry-run 和执行回滚计划。
- `metadata:submit`：提交 TMDB、IMDb、AI 等外部匹配请求或覆盖外部提交拦截记录。
- `audit:read`：查看审计日志。
- `task:manage`：查看任务治理页面。

## 4. 审计日志

审计日志用于回答“谁在什么时间对什么对象做了什么，结果是什么”。

系统会记录：

- 登录成功和登录失败。
- 用户维护。
- 系统设置保存。
- 媒体源新增、编辑和删除。
- 外部提交拦截覆盖处理。
- 真实重命名执行。
- 回滚计划创建、dry-run 和执行。

审计详情会自动脱敏 API Key、token、密码和 SMB 凭据。

## 5. 操作运行日志

操作运行日志用于查看扫描、重命名和回滚任务执行过程。

支持能力：

- 按任务查看日志。
- 自动刷新和手动刷新。
- 按级别筛选。
- 复制当前日志。
- 导出单任务文本日志。
- 日志过多时只保留最近展示内容，避免页面卡顿。
- 日志被清理后显示中文提示。

操作运行日志和审计日志职责不同：审计日志用于追责，操作运行日志用于排查任务过程。

## 6. 重命名回滚

回滚只能基于已成功的重命名批次创建。

建议流程：

1. 在重命名批次入口创建回滚计划。
2. 执行回滚 dry-run。
3. 确认当前文件存在、目标路径无冲突、目录可写。
4. 用户确认后执行回滚。
5. 查看回滚结果和回滚日志。

安全规则：

- 回滚不会覆盖已有文件。
- 回滚前必须 dry-run。
- 部分条目失败时，已成功条目保留，失败条目记录原因。
- 回滚后同步媒体文件路径、命名预览状态和增量扫描索引。

## 7. 任务治理

任务治理页面用于集中查看任务状态。

当前聚合：

- 扫描任务。
- 重命名批次。
- 回滚计划。

支持筛选：

- 任务类型。
- 状态。
- 媒体源。
- 时间范围。

列表中可以跳转到对应页面，也可以查看关联操作运行日志。

归档行为：

- 点击任务行的归档按钮后，该任务默认不再出现在任务治理列表。
- 归档只影响任务治理展示，不删除扫描任务、扫描结果、重命名批次、回滚计划或真实文件。
- 勾选“显示已归档”后可以查看已归档任务，并可执行恢复。

## 8. 当前限制

- M9 不提供企业级多租户、LDAP、OIDC、OAuth 或双因素认证。
- 任务归档入口已在 M9+ `0.9.1` 补齐；复杂自动归档策略仍属于后续增强。
- 远程协议 WebDAV、FTP、SFTP、S3 仍属于后续阶段。
- 回滚只能处理系统已记录的成功重命名，不强制恢复被外部程序移动、覆盖或删除的文件。

# MediaAI Renamer M1 用户手册

版本：v0.1.0 预览版  
阶段：M1 本地目录扫描与页面框架  
日期：2026-06-24

## 1. 当前可以做什么

M1 版本用于扫描本地或已挂载的影视目录，识别其中的视频文件，并在页面中查看媒体源、扫描任务、扫描结果和操作日志。

当前版本不会改动你的真实文件名，也不会执行重命名。

## 2. 访问地址

默认 Web 地址：

```text
http://localhost:8971
```

开发预览地址：

```text
http://127.0.0.1:5173
```

后端 API 默认端口：

```text
8970
```

## 3. 页面布局

页面分为左右两部分。

左侧是导航区：

- Logo 和产品名称。
- 媒体源菜单。
- 扫描任务菜单。
- 扫描结果菜单。
- 后端状态。
- 语言占位框。
- 版本号。
- 明暗主题按钮。
- 退出入口。

右侧是操作台：

- 点击左侧菜单后，右侧切换到对应页面。
- 右上角有全局搜索框，M1 只提供入口，搜索功能后续接入。

## 4. 侧栏展开和收缩

点击侧栏顶部的小三角可以展开或收缩菜单。

展开状态：

- 显示菜单图标和文字。
- 显示状态文字、版本号、主题按钮和当前用户。

收缩状态：

- 只保留图标。
- 状态用小圆点显示。
- 鼠标悬浮在图标上可以看到提示。

## 5. 主题切换

左下角版本号旁边有主题按钮。

点击后可在亮色和暗色之间切换。

暗色主题已经覆盖：

- 侧栏。
- 操作台。
- 表格。
- 表单。
- 输入框。
- 日志抽屉。
- 弹出层。

## 6. 后端状态

左下角状态点表示后端连接情况：

- 绿色：后端连接正常。
- 红色：后端连接异常。
- 灰色：状态未知或正在检测。

展开侧栏后，会显示状态文字。

## 7. 添加媒体源

进入“媒体源”页面。

填写：

- 名称：用于区分目录，例如“电影”或“电视剧”。
- 目录路径：本机或已挂载目录路径。
- 状态：启用或停用。

点击“保存媒体源”后，媒体源会保存到系统中。

注意：

- 目录路径不能为空。
- 目录必须存在。
- 路径必须是目录，不是单个文件。

## 8. 启动扫描

进入“扫描任务”页面。

操作步骤：

1. 在媒体源选择框中选择一个已保存的媒体源。
2. 点击“开始全量扫描”。
3. 等待扫描完成。
4. 在任务表格中查看扫描结果统计。

任务表格会显示：

- 任务 ID。
- 状态。
- 已扫描数量。
- 视频文件数量。
- 警告数量。
- 批大小。
- 批间隔。
- 开始时间。
- 结束时间。

## 9. 扫描规则

M1 使用分批扫描。

默认设置：

- 每批 100 个文件。
- 每批之间间隔 1 秒。

支持识别的视频格式：

`.mp4`、`.mkv`、`.avi`、`.mov`、`.wmv`、`.flv`、`.m4v`、`.ts`、`.m2ts`、`.rmvb`、`.mpg`、`.mpeg`、`.webm`

扩展名大小写不影响识别。

## 10. 查看扫描结果

进入“扫描结果”页面。

表格会显示已识别的视频文件：

- 文件名。
- 格式。
- 文件大小。
- 任务 ID。
- 文件路径。
- 修改时间。

M1 只支持查看结果，不支持命名预览和文件重命名。

## 11. 查看操作日志

进入“扫描任务”页面。

点击“查看日志”。

日志会以抽屉形式弹出。

支持操作：

- 刷新日志。
- 导出 TXT。

## 12. 退出入口

左下角显示退出入口和当前用户。

当前默认用户为：

```text
admin
```

M1 还没有用户登录和权限系统，因此点击退出入口暂不执行真实退出逻辑。

## 13. 常见问题

### 13.1 保存媒体源失败

请检查：

- 路径是否为空。
- 路径是否存在。
- 路径是否是目录。
- 当前程序是否有权限访问该目录。

### 13.2 扫描后没有结果

请检查：

- 媒体源目录中是否有支持的视频格式。
- 文件扩展名是否属于 M1 支持列表。
- 媒体源是否选对。
- 扫描任务是否已完成。

### 13.3 后端状态异常

请检查：

- 后端服务是否启动。
- 后端端口是否为 `8970`。
- 页面是否可以访问 `/api/health`。

### 13.4 暗色主题下有颜色异常

请刷新页面。如果仍然存在，请记录出现问题的页面和控件位置，后续统一优化。

## 14. 当前限制

M1 不支持：

- 搜索实际结果。
- 语言切换。
- 用户登录和权限。
- 元数据匹配。
- AI 识别。
- 命名预览。
- 真实重命名。
- 增量扫描。

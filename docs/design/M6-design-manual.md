# MediaAI Renamer M6 设计手册

版本：`0.6.9`
覆盖范围：AI 智能解析、敏感词设置、外部提交保护
更新日期：2026-07-03

## 1. 阶段目标

M6 在 M5 的 NAS / SMB / NFS 共享目录能力基础上，补齐 DeepSeek / OpenAI-compatible AI 智能解析能力，并先建立外部提交保护边界。

核心目标：

- 用户确认前不调用 AI。
- 文件名、路径或解析标题命中敏感词时，不提交 TMDB、IMDb、AI 或后续外部刮削源。
- AI 结果只作为结构化候选展示，不能直接重命名真实文件。
- 用户确认 AI 候选后，只回填命名预览，仍需走 dry-run 和安全重命名。

## 2. 后端设计

### 2.1 AI Provider

后端新增统一 AI Provider 抽象，首期实现 DeepSeek Provider。Provider 使用 OpenAI-compatible `/chat/completions` 接口，返回：

- 调用状态。
- 文本内容。
- 响应耗时。
- token usage。

异常处理要求：

- 未配置 API Key 时返回中文错误。
- 网络、超时、鉴权失败和服务端错误均降级为可读结果。
- 不记录完整 prompt、API Key 或敏感路径。

### 2.2 结构化解析

AI 解析服务输入：

- 文件名。
- 文件路径。
- 本地解析出的标题、类型、年份、季号和集号。

AI 输出必须是 JSON 对象，字段包括：

- `title`
- `media_type`
- `year`
- `season`
- `episode`
- `confidence`
- `reason`

后端负责二次校验字段类型和范围；非法 JSON、空标题或范围异常不回填，只返回失败状态。

### 2.3 外部提交保护

外部提交保护统一在后端执行，前端提示不能作为安全边界。

保护输入范围：

- 原文件名。
- 完整路径。
- 解析标题。
- 用户选择用于匹配的标题字段。

命中规则时：

- 不调用 TMDB、IMDb 或 AI Provider。
- 写入 `external_submission_blocks` 拦截记录。
- 返回可读提示，用户可在敏感词设置页查看和处理。

## 3. 前端设计

### 3.1 敏感词设置

系统设置中新增“敏感词设置”页，包含：

- 启用默认敏感词库。
- 默认影视风险词编辑。
- 自定义敏感词编辑。
- 解析结果预览。
- 外部提交拦截记录列表。

敏感词使用 `||` 分隔，加载历史换行数据时会统一转换为 `||` 展示。

### 3.2 AI 智能解析配置

系统设置中新增“AI 智能解析”分类，包含：

- 启用开关。
- 服务商。
- 模型名。
- API Key。
- Base URL。
- 接口超时时间。
- 最大重试次数。
- 测试连接。

测试连接状态栏显示完整时间格式 `yyyy-mm-dd hh:mm:ss`。

### 3.3 重命名页 AI 解析

重命名预览列表支持单条 AI 解析：

- 行操作点击“AI 解析”。
- 后端返回候选弹窗。
- 用户点击“选择回填”后更新命名预览目标文件名。
- 回填后仍需用户手动执行重命名。

待处理文件列表支持单条 AI 解析：

- 行操作点击“AI 解析”。
- 弹窗展示候选结果。
- 当前阶段不直接从待处理文件生成命名预览，不提供回填按钮。

## 4. 回填规则

AI 候选回填复用 TMDB 元数据回填链路：

- 先读取当前命名构建器模板。
- 模板中存在的字段如果候选有值，则写入目标文件名。
- 候选缺少模板必需字段时，保留本地解析已有字段。
- 电影候选缺少年份时，如果电影模板包含年份，则保留本地年份。
- 剧集候选缺少季号或集号时，如果剧集模板包含季集字段，则保留本地季集。

## 5. 安全边界

M6 不做：

- 自动批量调用 AI。
- 扫描时自动提交 AI。
- AI 直接执行真实重命名。
- 多模型调度、费用估算和本地模型部署。

真实文件修改仍只发生在 M3 已有安全重命名流程中。

## 6. 验证范围

M6 自动化验证覆盖：

- AI 配置保存和脱敏。
- AI 连接测试。
- DeepSeek Provider mock。
- AI 结构化解析成功、失败和非法输出。
- 敏感词阻断 TMDB 和 AI 调用。
- 外部提交拦截记录列表和处理。
- 重命名预览 AI 候选回填。
- 待处理文件 AI 解析。
- 前端 API 客户端和构建。

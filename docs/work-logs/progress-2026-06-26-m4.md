# 2026-06-26 M4 开发启动

## 范围确认

- 用户提供新版 M4 开发文档与验收清单，目录为 `docs/development/m4/`。
- M4 按新版计划全量开发，并拆分为 `M4-1` 到 `M4-4` 四个小版本。
- 当前先处理前置风险，再启动 `M4-1` 统一热配置中心。

## 前置风险修正

- 修复后端 `AppSettings.version` 仍为 `0.3.0` 的问题，更新为 `0.3.1`。
- 更新健康检查测试，确保 `/api/health` 返回版本与发布版本一致。
- 新增轻量数据库迁移框架，通过 `app_meta.schema_version` 记录 schema 版本。
- 当前 schema 版本为 4，服务启动时会为旧库补齐 `system_settings` 表和 `rename_previews` 的 M4 元数据字段。
- 新增迁移测试，覆盖旧 M3 数据库升级和新数据库记录当前 schema version。

## M4-1 已完成内容

- 新增 `system_settings` 数据表。
- 新增系统热配置 schema、服务和 API。
- 支持配置优先级：环境变量 > 数据库热配置 > 默认值。
- 支持 TMDB 配置默认值、校验、敏感值脱敏和热读取。
- 新增前端设置页、设置菜单、路由和 Pinia store。
- 设置页采用左侧分类菜单 + 右侧配置表单，当前落地 TMDB 元数据刮削配置，其余分类为占位。

## M4-2 已完成内容

- 新增 TMDB 元数据候选模型、匹配结果模型和匹配汇总模型。
- 新增 100 分加权匹配算法：标题相似度、年份、类型、季集维度共同评分。
- 新增 TMDB 搜索服务编排层，支持未启用、未配置密钥、接口异常时降级。
- 新增 Mock 友好的 provider 接口，业务测试不依赖真实外网。
- 新增 TMDB 客户端骨架，支持按电影/剧集搜索 TMDB 候选。

## M4-3 已完成内容

- 命名预览返回 TMDB 来源、匹配状态、匹配分数和匹配消息字段。
- 新增单条命名预览 TMDB 匹配 API。
- 新增候选列表 API 和人工选择候选回填 API。
- 高置信结果自动回填目标文件名；低置信结果保持需确认并提供候选弹窗。
- 前端命名预览页新增 TMDB 匹配图标按钮、元数据状态列、匹配度列和候选选择弹窗。

## M4-4 已完成内容

- 新增 `pending_files` 待处理文件表，schema version 提升到 5。
- 扫描阶段读取 `scan.minimum_file_size` 热配置，小于阈值的视频文件进入待处理队列，不进入普通扫描结果。
- 新增待处理文件服务，支持列表、单条移除、清空和批量迁移文件。
- 新增待处理文件 API。
- 命名预览页新增待处理文件列表模块，支持查看、单条移除、清空列表和批量迁移入口。

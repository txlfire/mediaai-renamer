## 追加进展：共享协议校验闭环

- 协议接口新增媒体源上下文，连接测试、目录浏览、扫描前校验、dry-run 和真实重命名前校验可以读取媒体源路径类型、SMB 账号状态、NFS 说明字段等配置。
- SMB / UNC 连接测试从“仅校验路径格式”升级为“路径格式 + 当前服务进程可访问性 + 读写权限”校验；不可达、无权限或未登录共享时返回中文错误和排查建议。
- 编辑媒体源弹窗的测试连接已使用页面当前填写的账号、密码、NFS 字段进行测试，不再只传路径。
- mounted_nfs 连接测试在 Linux 环境下会尽量读取 `/proc/mounts` 识别文件系统类型；无法确认时仅给出提示，不在路径可访问时强制失败。
- 本地路径检测捕获 `OSError` / `PermissionError`，共享路径不可达时不再让后端异常崩出。
- M5 验收清单已同步勾选本轮完成的连接测试、目录浏览权限、扫描前校验、重命名前校验相关条目。

## 未完成与风险

- 外部 NAS / SMB / NFS 实机验收仍依赖目标部署环境，需要在真实 Windows 共享、Linux / NAS / Docker 挂载路径中逐项验证。
- 部分系统设置项已支持保存和读取，但尚未全部接入实际业务行为，例如日志保留策略和 NFS 超时配置的底层强制中断。
- M5 不提供自动挂载、`net use` 登录、NFS Kerberos 或跨设备迁移能力。

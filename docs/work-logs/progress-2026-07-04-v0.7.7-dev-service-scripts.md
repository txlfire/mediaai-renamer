# 2026-07-04 v0.7.7 开发服务脚本整理

## 本次范围

- 新增 Windows 独立开发服务入口脚本：`scripts/dev-windows.ps1`。
- 新增 Linux/macOS 独立开发服务入口脚本：`scripts/dev-linux.sh`。
- 新增 Docker Compose 独立服务入口脚本：`scripts/dev-docker.sh`。
- 新增 Windows 前端后台静态代理服务：`scripts/serve-frontend.py`。
- 新增 npm 调用入口：
  - `npm.cmd run dev:windows -- start|stop|restart|status`
  - `npm run dev:linux -- start|stop|restart|status`
  - `npm run dev:docker -- start|stop|restart|down|status|logs`
- 更新 `scripts/README-zh.md`，补充三个入口脚本的用途、动作和关键步骤。

## 设计说明

- 操作系统入口分开，避免 Windows、Linux、Docker 参数和依赖混用。
- Windows/Linux 本地开发入口复用现有启停脚本，降低行为分叉。
- Windows 后台前端服务改为托管 `frontend/dist` 并代理 `/api`，避免 Vite CLI 在后台进程 stdin 或父进程退出后停止。
- Docker 入口直接封装 `docker compose`，默认使用源码构建配置 `docker-compose.yml`，可通过 `COMPOSE_FILE=docker-compose.ghcr.yml` 切换到镜像部署配置。
- Windows 状态检查使用 HTTP 探活并增加 TCP 兜底，避免端口查询受权限或地址族影响导致误报。

## 验证

- `npm.cmd run dev:windows -- restart`：已在沙箱外执行。
- 后端健康检查：`http://127.0.0.1:8970/api/health` 返回 `0.7.7 ok`。
- 前端访问检查：`http://127.0.0.1:5173` 返回 `200`。
- 前端代理检查：`http://127.0.0.1:5173/api/health` 返回 `0.7.7 ok`。
- `npm.cmd run dev:windows -- status`：显示前后端 running。
- `npm.cmd run check:encoding`：通过。

## 限制

- 当前 Windows 环境没有可用 WSL 发行版，`bash -n` 无法校验 Linux/Docker shell 脚本。
- 当前 Windows 环境未安装 Docker CLI，无法执行 `docker compose config`。

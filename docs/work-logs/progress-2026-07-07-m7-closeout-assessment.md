# 2026-07-07 M7 阶段收尾评估

## 结论

- M7 主体能力已经基本落地，但**当前不能判定为完全收尾**，不建议直接按正式版本发布。
- 现有 `0.7.9` 变更更接近 **M7 收尾候选版本**，还差少量功能闭环和阶段文档收口。

## 已确认完成的范围

- M7-0 数据结构与迁移：
  - `scan_file_index` 表、`scan_jobs` 增量统计字段、旧接口 `scan_mode` 默认兼容。
- M7-1 增量扫描后端：
  - 全量写索引、增量跳过未变化文件、统计新增/变更/缺失/跳过。
- M7-2 重命名联动：
  - 重命名成功后源路径标记 `renamed`，目标路径写为 `active`。
  - 文件名标题不足时支持上级文件夹标题兜底。
  - 支持手动排除到待处理列表。
- M7-3 / M7-3.5 前端与匹配链路：
  - 扫描任务页支持全量/增量选择与建议模式提示。
  - 重命名页支持批量 AI、TMDB→AI、Provider 管理、批量执行门禁、确认提示和 Token 汇总。

## 当前剩余工作

### 1. 标题识别模式未完整闭环

设计文档要求支持以下四种模式：

- `file_name_first`
- `parent_folder_fallback`
- `parent_folder_first`
- `manual_only`

当前实现状态：

- 仅实际落地了默认 `parent_folder_fallback` 行为。
- `recognition_mode` 字段已入库，但设置页、服务层和重新生成预览链路尚未完整支持四种模式切换。

### 2. 标题冲突与详情展示未完全闭环

- 文件名与上级文件夹标题冲突时，目前不会自动覆盖文件名标题，但**尚未完整标记冲突原因并在详情中完整展示**。
- 预览详情当前可展示标题来源和上级文件夹标题，但**未完整展示识别模式和冲突原因**。

### 3. 外部匹配来源选择未完全覆盖

- TMDB 匹配目前支持 `原文件名 / 解析标题` 两种来源。
- 设计中提到的“上级文件夹标题作为独立匹配来源”尚未完整开放为可选项。

### 4. M7 里程碑文档仍缺一份独立用户手册

- 已更新完整用户手册 `docs/manuals/MediaAI-Renamer-用户手册.md`。
- 但按本项目里程碑文档惯例，`docs/manuals/` 下仍缺少一份独立的 M7 用户手册。

### 5. 回归项尚未作为 M7 阶段结论单独勾闭

- 后端测试、前端测试、前端构建、编码检查已经通过。
- 但 M7 验收清单中的 M1-M6 回归项仍未形成单独的阶段结论，不宜直接宣称 M7 已完全验收。

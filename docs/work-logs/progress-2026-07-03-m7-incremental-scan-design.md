## 设计结论

- 增量扫描不计算全文件哈希，默认使用 `media_source_id + normalized_path + file_size + modified_at + extension` 作为轻量指纹，避免 NAS / SMB / NFS 场景下高 IO。
- 新增 `scan_file_index` 保存文件索引快照，扫描任务按模式写入新增、修改、跳过、删除或失效统计。
- 旧 API 默认保持全量扫描，保证兼容现有扫描入口。
- 真实重命名成功后必须更新扫描索引，避免旧路径在后续增量扫描中被误判为删除和新增。

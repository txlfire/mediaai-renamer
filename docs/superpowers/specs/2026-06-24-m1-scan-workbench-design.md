# M1 本地目录扫描与页面框架设计

## 1. 目标

M1 的目标是让 MediaAI Renamer 能扫描本地或已挂载媒体目录，识别视频文件，将扫描结果写入 SQLite，并在页面上查看媒体源、任务状态、扫描结果和操作日志。

同时，M1 建立后续 M2/M3 继续复用的主界面框架：左侧可伸缩菜单，右侧操作台，左下全局状态区和主题切换。

## 2. 非目标

M1 不实现以下能力：

- TMDB、Bangumi、TVDB、豆瓣等外部元数据匹配。
- DeepSeek 或其他 AI 解析。
- 增量扫描。
- 标准命名预览。
- 真实文件重命名。
- 用户登录、权限、会话管理。

退出按钮在 M1 只保留界面入口，不执行真实用户退出逻辑。

## 3. 前端布局

### 3.1 主框架

页面采用左右布局：

- 左侧：应用侧栏。
- 右侧：操作台区域。

侧栏支持展开和收起：

- 展开状态显示 Logo、标题、菜单图标和菜单文字。
- 收起状态保留 Logo 简写、菜单图标和必要状态提示。

右侧操作台不改变整体页面框架，只根据当前菜单切换业务界面。

### 3.2 左侧顶部

左侧顶部展示：

- Logo 区域。
- 标题：`MediaAI Renamer`。
- 菜单折叠按钮。

### 3.3 左侧菜单

M1 初版菜单包含：

- 媒体源：配置本地或已挂载目录。
- 扫描任务：启动全量扫描并查看任务状态。
- 扫描结果：查看识别出的视频文件列表。

后续阶段再加入：

- 命名预览。
- 安全重命名。
- AI 解析。
- 系统设置。

### 3.4 左侧底部

左侧底部展示全局信息和全局操作：

- 提示语：例如“先扫描目录，再生成命名预览”。
- 系统状态：后端连接正常、异常或检测中。
- 软件版本：来自 `/api/health` 返回值，初始值为 `v0.1.0`。
- 主题切换：跟随系统、亮色、暗色。
- 退出按钮：M1 仅展示入口，后续接入用户权限后启用。

主题切换放在左下区域，不放入右侧系统设置页面。

## 4. 主题切换

M1 支持三种主题模式：

- 跟随系统。
- 亮色。
- 暗色。

默认模式为跟随系统。

前端将主题模式保存在浏览器本地存储中。页面启动时读取本地设置：

- 如果为亮色或暗色，直接应用对应主题。
- 如果为跟随系统，使用 `prefers-color-scheme` 判断系统主题。

当系统主题变化且当前模式为跟随系统时，页面应自动切换。

主题作用范围为整个前端应用，包括侧栏、操作台、表格、弹窗、按钮和日志查看弹窗。

## 5. 操作日志

操作日志不作为左侧常驻菜单。

M1 提供日志弹出页：

- 从扫描任务页或顶部工具区域打开。
- 展示扫描相关日志。
- 支持按日志类型或级别筛选。
- 支持导出 TXT。

弹出形式优先使用抽屉；如果当前布局空间不足，可使用对话框。日志弹出页关闭后不影响当前操作台页面。

## 6. 后端能力

### 6.1 媒体源

后端需要保存媒体源目录信息，包括：

- 目录名称。
- 目录路径。
- 启用状态。
- 创建时间。
- 更新时间。

路径必须进行基础校验：

- 路径不能为空。
- 路径必须存在。
- 路径必须是目录。
- 路径不可访问时返回明确错误。

M1 不做共享协议挂载，不保存 SMB/NFS 凭据。

### 6.2 视频格式识别

M1 识别常见视频扩展名：

- `.mp4`
- `.mkv`
- `.avi`
- `.mov`
- `.wmv`
- `.flv`
- `.m4v`
- `.ts`
- `.m2ts`
- `.rmvb`
- `.mpg`
- `.mpeg`
- `.webm`

扩展名大小写不敏感。

### 6.3 全量扫描

M1 支持全量扫描：

- 从媒体源根目录递归扫描。
- 采用分批扫描，避免单次扫描占用过多资源。
- 跳过不可访问目录并记录警告。
- 识别视频文件并写入 SQLite。
- 单文件读取失败不终止整体扫描。
- 系统级错误会终止任务并记录错误。

分批扫描配置：

- 每批扫描数量可配置。
- 默认每批扫描 `100` 个文件。
- 每批之间的间隔时间可配置。
- 默认批间隔为 `1` 秒。
- 配置值必须在后端启动时完成校验，批大小必须大于 0，批间隔不得小于 0。
- 扫描任务记录应保存实际使用的批大小和批间隔，便于后续排查任务行为。

扫描结果至少包含：

- 文件路径。
- 文件名。
- 扩展名。
- 文件大小。
- 最后修改时间。
- 所属媒体源。
- 扫描任务 ID。

### 6.4 扫描任务状态

扫描任务状态至少包含：

- `pending`：已创建，等待执行。
- `running`：执行中。
- `completed`：执行完成。
- `failed`：系统级失败。

任务记录至少包含：

- 任务 ID。
- 媒体源 ID。
- 状态。
- 批大小。
- 批间隔秒数。
- 已扫描文件数。
- 已识别视频数。
- 警告数。
- 错误信息。
- 开始时间。
- 结束时间。

M1 可以先采用同步 HTTP 触发扫描。扫描耗时较长时，后续再升级为后台任务或 WebSocket 实时进度。

## 7. 数据库

M1 新增表：

- `media_sources`：保存媒体源目录。
- `scan_jobs`：保存扫描任务。
- `media_files`：保存扫描识别结果。

数据库初始化仍集中在 `app.core.database`，业务层不得重复创建表。

## 8. API

M1 API 均使用 `/api` 前缀。

建议接口：

- `GET /api/media-sources`
- `POST /api/media-sources`
- `PUT /api/media-sources/{source_id}`
- `DELETE /api/media-sources/{source_id}`
- `POST /api/scan-jobs`
- `GET /api/scan-jobs`
- `GET /api/scan-jobs/{job_id}`
- `GET /api/media-files`
- `GET /api/logs`
- `GET /api/logs/export`

API 层只处理请求和响应，扫描、路径校验和数据库读写放在 service 层。

## 9. 前端页面

### 9.1 媒体源页面

用于添加、编辑、启用和删除媒体源目录。

表单字段：

- 名称。
- 路径。
- 是否启用。

提交前使用 Element Plus 表单校验。

### 9.2 扫描任务页面

用于选择媒体源并启动全量扫描。

展示：

- 任务状态。
- 已扫描文件数。
- 已识别视频数。
- 警告数。
- 开始时间。
- 结束时间。

提供“查看日志”按钮打开日志弹出页。

### 9.3 扫描结果页面

用于查看扫描出的媒体文件。

展示：

- 文件名。
- 所属媒体源。
- 文件路径。
- 扩展名。
- 文件大小。
- 最后修改时间。
- 扫描任务 ID。

M1 只查看结果，不提供命名预览和重命名操作。

## 10. 工作日志目录

所有项目工作日志统一存放在：

```text
docs/work-logs/
```

该目录保存：

- 每日进度记录。
- 阶段复盘。
- 阶段启动记录。
- 验证结果记录。

运行时程序日志继续保存在：

```text
logs/
```

两类日志职责不同，不混用。

## 11. 默认端口

系统默认对外 Web 访问端口为 `8971`。

Docker Compose 部署时：

- 前端 Nginx 对外暴露 `8971`。
- 后端默认端口为 `8970`，容器内部和宿主机映射均使用 `8970`。
- 前端通过同源 `/api` 访问后端，不在页面代码中硬编码后端主机和端口。

本地开发端口可以继续使用 Vite 默认开发端口，但文档中应明确生产和 NAS 部署入口为：

```text
http://localhost:8971
```

## 12. 测试与验收

后端验收：

- 可以新增媒体源。
- 无效路径会返回明确错误。
- 可以扫描测试目录并识别视频文件。
- 扫描按配置分批执行，默认批大小为 100，默认批间隔为 1 秒。
- 扫描结果写入 SQLite。
- 单个不可读文件或目录不会中断整个扫描。

前端验收：

- 左侧菜单可切换媒体源、扫描任务、扫描结果。
- 侧栏可展开和收起。
- 左下展示系统状态、版本、主题切换和退出按钮。
- 主题支持跟随系统、亮色和暗色。
- 操作日志通过弹出页查看。
- 部署默认访问端口为 `8971`。

验证命令：

```powershell
$env:PYTHONPATH = "backend"
.\.venv\Scripts\python.exe -m unittest discover backend\tests -v
npm run frontend:test
npm run frontend:build
```

Docker 相关变更需要在具备 Docker 的环境中额外执行：

```powershell
docker compose up --build
```

# M6 AI 智能解析验收清单

验收结论：`0.6.9` 已完成 M6 核心验收范围。AI 调用必须由用户主动触发；命中敏感词时后端会阻断 TMDB / IMDb / AI 外部提交并记录拦截；重命名预览支持 AI 候选确认回填；待处理文件支持 AI 候选展示。批量 AI 解析按开发计划仍为 M7 规划范围，不在 M6 正式版执行。

最近验证：

- `npm.cmd run backend:test`：155 个测试通过。
- `npm.cmd run frontend:test`：11 个测试文件、55 个测试通过。
- `npm.cmd run frontend:build`：通过，仅保留既有 Vite chunk 体积提示。
- `npm.cmd run check:encoding`：通过。

## 0. 敏感词过滤验收

- 系统内置默认影视内容风险敏感词，覆盖可能在 DeepSeek、TMDB、IMDb 或后续刮削网站中引发风险的黄色、暴力、血腥、毒品等影视内容词汇。
- 用户可在系统设置中选择是否启用默认敏感词库，并可编辑默认敏感词列表，保存后无需重启即可生效。
- 用户可在系统设置中新增、编辑、删除自定义敏感词，保存后无需重启即可生效。
- 原文件名、完整路径、解析标题或用户选择用于匹配的标题字段命中敏感词时，TMDB、IMDb 和后续其他刮削源不得发出外部请求。
- 命中敏感词时，AI 解析不得发出外部请求。
- 命中敏感词时，本地解析、命名预览、人工编辑、dry-run 和安全重命名仍可继续使用。
- 页面必须展示明确提示，说明该条目已阻止外部提交，并给出命中的敏感词或脱敏后的命中原因。
- 批量操作中命中敏感词的条目按跳过处理，结果摘要中展示跳过数量和原因。
- 被敏感词或其他规则拦截的条目必须写入拦截记录，并能在“外部提交拦截列表”中查询。
- 拦截列表必须展示文件名、路径、来源模块、拦截规则、命中内容脱敏摘要、原计划提交目标、拦截时间和处理状态。
- 用户可以对拦截记录选择“修改后重新匹配”“不进行匹配并忽略”“无视规则继续提交”。
- 选择“修改后重新匹配”后，应基于修改后的文件名或解析标题重新执行规则检查；仍命中规则时不得提交外部服务。
- 选择“不进行匹配并忽略”后，记录状态更新为忽略，不再进入默认待处理提醒，但历史仍可查询。
- 选择“无视规则继续提交”时，必须弹出强确认提示，说明可能暴露文件名、路径或解析标题，用户确认后才允许提交。
- 覆盖提交必须记录用户决策、覆盖原因、确认时间、操作者和提交目标；覆盖只对当前记录和当前目标服务生效。
- 自动化测试必须验证默认影视内容风险敏感词、默认词库开关、默认词库修改、自定义敏感词、英文大小写匹配、TMDB 阻断、AI 阻断和未命中时正常调用。
- 自动化测试必须验证拦截记录写入、列表查询、忽略状态、修改后重新匹配和覆盖提交审计记录。

## 1. 配置验收

- 系统设置中存在 AI 智能解析分类。
- 系统设置中存在“敏感词设置”配置区域，用于维护默认敏感词开关、默认敏感词列表和自定义敏感词。
- 支持启用开关、服务商、模型名、API Key、Base URL、超时时间、重试次数配置。
- 配置保存后无需重启服务即可生效。
- API Key 前端脱敏展示，后端不返回明文。
- 未配置或关闭 AI 时，系统继续使用本地解析、共享目录扫描和 TMDB 流程。
- 连接测试成功时返回中文成功提示，失败时返回中文失败原因。

## 2. 后端服务验收

- AI Provider 接口可替换，DeepSeek 为首个实现。
- 测试不依赖真实外网，使用 mock provider 覆盖成功、失败、超时和非法输出。
- AI 调用失败不抛致命异常，不中断现有任务。
- 日志记录模型名、耗时、token 数和状态，不记录完整 prompt、API Key 或敏感路径。
- 返回结构必须经过 schema 校验。

## 3. 解析结果验收

- AI 能返回标题、类型、年份、季、集、置信度和说明。
- 年份、季、集必须为合法数字范围。
- 类型必须映射到系统已有类型。
- 空标题、非法字符、明显冲突结果不得自动回填。
- 通过校验的结果作为候选展示，由用户确认后回填命名预览。

## 4. 前端交互验收

- 待处理文件列表可对单条记录发起 AI 解析。
- 命名预览页可对低置信或未匹配记录发起 AI 辅助解析。
- AI 解析中有加载状态，重复点击被拦截。
- 成功后打开候选确认弹窗。
- 失败后展示中文错误，不清空原有数据。
- 用户确认候选后，目标文件名更新到命名预览，不直接重命名真实文件。

## 7. 追加验收：元数据回填与列表稳定性

- 选择 TMDB、IMDb 或后续 AI 候选回填时，必须读取当前命名构建器模板并按模板变量合并字段。
- 当候选缺少当前模板需要的字段时，必须保留本地解析已有字段；例如剧集模板包含季集组合时，电视剧候选缺少季/集不得清空原 `SxxExx`，电影模板包含年份时，电影候选缺少年份不得清空原年份。
- 回填后的目标文件名必须重新按当前命名构建器生成，不能绕过用户配置。
- 元数据状态、匹配度或详情中必须展示匹配原因；低置信结果必须提示需要人工确认。
- 重命名预览刷新后默认排序必须稳定，不能因为 `updated_at` 更新导致当前页记录跳转到其他页；用户主动排序除外。
- 自动化测试必须覆盖电影和剧集候选缺字段时保留模板必需字段的回填场景。

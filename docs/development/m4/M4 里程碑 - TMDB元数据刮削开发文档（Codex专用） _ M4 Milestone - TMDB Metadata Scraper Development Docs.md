## 三、统一系统设置中心 \+ 全局热部署架构（核心底座）

### 3\.1 页面架构规范（前端强制）

系统唯一全局配置入口，采用**左侧分类菜单 \+ 右侧配置表单** 布局，模块化可拓展，后续所有系统配置统一收敛至此页面。

本次新增分类：【TMDB 元数据刮削配置】，其余分类仅预留架构占位。

### 3\.2 全局热部署技术规范（Codex 架构统一标准）

#### 3\.2\.1 四层运行机制

- **存储层**：所有热配置结构化持久化数据库，字段包含：分类、配置Key、配置值、备注、更新时间、操作人。废弃静态配置文件、硬编码常量。

- **缓存层**：内存常驻缓存 \+ 配置变更主动刷新 \+ 定时兜底刷新，保障秒级热生效，杜绝缓存僵死。

- **校验层**：保存前强制后端参数校验，非法值直接驳回，不入库、不更新缓存、不生效。

- **降级层**：配置读取异常、缓存失效、参数非法时，自动加载代码预设默认值，业务不中断、不报错。

#### 3\.2\.2 三级配置优先级（固定不可修改）

**环境变量 \> 数据库热配置 \> 系统默认兜底值**

- 环境变量存在值则优先生效（用于生产密钥隔离、容器部署配置）

- 环境变量为空，读取页面数据库配置

- 无配置/配置异常，启用代码默认值兜底

#### 3\.2\.3 安全强制规则

- 敏感密钥（TMDB Key）前端全程脱敏星号展示，仅支持覆写，不回显明文

- 全局日志自动脱敏，禁止输出密钥明文

- 配置新增/修改接口仅管理员权限可访问

- 所有配置变更留存操作日志（操作人、时间、新旧值）

### 3\.3 全局统一参数校验规则（Codex 强制执行）

本小节汇总**全项目唯一、最终、最全的参数校验规则总表**，Codex 开发所有配置必须严格遵循，无特例、无自定义规则。所有校验失败统一逻辑：驳回保存、不入库、不刷新缓存、业务不中断、自动加载默认兜底值。

#### 3\.3\.1 通用基础参数校验规则（全局通用）

- **超时类参数**：必须为正整数，合法区间 1000ms \~ 30000ms，超限直接驳回

- **语种/地区参数**：严格枚举白名单校验，非法字符、不在枚举列表内直接驳回

- **密钥类参数**：允许空值（关闭对应功能），非空时校验合法字符格式，杜绝非法乱码字符

- **数值类参数**：并发数、分页条数、重试次数、文件最小扫描阈值等，必须为非负整数，支持上限约束，小数/负数/字符全部驳回

- **开关类参数**：仅支持布尔 true/false，其他任意字符、数字、空字符全部驳回

#### 3\.3\.2 个性化命名拖拽配置专属校验规则（M4\-Future 预留、强制架构校验）

- **1\. JSON 结构体合法性校验**：整体配置必须为标准合法JSON对象，必须包含顶层字段「separator」「fieldList」；JSON解析异常、字段缺失、格式错误直接驳回保存，系统自动兜底加载内置默认模板，不影响业务。

- **2\. 分隔符白名单校验**：仅允许四种合法分隔符：英文点 \`\.\`、下划线 \`\_\`、横杠 \`\-\`、空格；前端仅提供固定下拉选择，禁止自定义输入，非法字符直接拦截驳回。

- **3\. 字段完整性与锁定校验**：fieldList 必须永久包含 title 字段，title 的 locked 值强制为 true、enable 强制为 true、sort 强制为1；后端强校验，禁止删除、隐藏、解锁、拖拽换位标题字段，拦截所有非法篡改。

- **4\. 字段唯一性校验**：fieldList 内所有 key 全局唯一，不允许重复字段、空key、非法未知key；自动过滤未知冗余字段、垃圾参数，保证配置结构体纯净合规。

- **5\. 排序权重 Sort 校验**：sort 必须为正整数、全局唯一、无重复、无空值、无负数；前端拖拽完成后自动重整连续有序序号，后端二次校验排序合法性，非法排序直接驳回。

- **6\. 字段状态类型校验**：所有字段 enable、locked 仅支持布尔 true/false 类型；除 title\.locked=true 固定锁定，其余所有字段（含 titleEn）locked 强制为 false，后端拦截所有锁定状态篡改行为；双语标题字段参数格式、空值逻辑统一纳入强校验体系。

- **7\. 空配置与异常兜底校验**：数据库配置为空、结构体损坏、校验失败时，不报错、不阻塞，自动加载系统默认命名模板，保障重命名流程稳定可用。

- **8\. 重置参数合规校验**：点击「恢复默认」后，后端强制覆盖为标准默认结构体、默认分隔符、默认排序、默认字段状态，重置后自动触发缓存热刷新，参数全程合规校验通过后入库。

#### 4\.3\.0\.2 个性化命名配置补充规则

- **前端交互规则**：采用标准拖拽组件实现字段自由排序，支持实时拖拽预览、一键重置为默认模板，操作无刷新，搭配开关组件控制单字段显隐。

## 五、TMDB 刮削核心业务开发规范

### 5\.1 支持媒体类型

电影、剧集 两类资源完整支持。

### 5\.2 入参与出参规范

入参：本地解析结果（标题、年份、媒体类型）

出参固定结构：

- 刮削来源站点标识（TMDB/其他拓展站点）

- 站点候选结果列表（多站聚合）

- 单条结果**匹配度百分比分值（0%\-100%）**

- 整体匹配状态枚举

- 失败/异常站点原因枚举

### 5\.3 状态枚举定义（Codex 前后端统一）

#### 5\.3\.1 匹配状态

- 未匹配（匹配度 0%）

- 高置信度自动匹配（匹配度 ≥85%）

- 低置信度待人工选择（匹配度 30% \- 84%）

- 匹配失败（匹配度 0%）

#### 5\.3\.2 失败原因

- 未配置 TMDB API Key

- 接口请求超时

- 无匹配搜索结果

- TMDB 服务异常/限流

### 5\.4 降级容错强制逻辑

- 无 Key / Key 非法：关闭刮削能力，完全走本地解析，业务不中断

- 超时/限流/服务异常：记录错误原因，保留本地数据，不阻塞任务

- TMDB 所有逻辑为**增强功能**，不影响存量核心流程

- 读取原始文件名，执行存量完整逻辑：剧集季集识别 → 电影年份识别 → 分辨率/编码/画质噪声词清洗 → 生成本地建议名

- 本地解析完成后，二次筛选无效文件：识别信息残缺、纯片段/花絮/预告片、无有效匹配特征的文件，统一归入【无需刮削/待确认列表】

- 判断条件：任意已启用刮削站点密钥合法、网络正常

- 各站点独立请求、独立异常捕获，单站故障不阻塞整体流程

#### 5\.6\.6 匹配度加权算法完整落地规范（Codex 可直接编码 \[M4\-Online\]）

基于本地解析结果与TMDB标准元数据做**多维度加权比对**，输出 0\-100% 标准化匹配分值。分值仅用于预览置信展示、自动/手动匹配决策，**不侵入、不改动原有本地解析、重命名存量逻辑**。所有计分无负分、0分保底、100分封顶，电影/剧集维度自适应适配。

**固定权重分配（不可修改）**

- 标题文本相似度：60分（核心权重，中英双语标题择优取最高分）

- 年份匹配精度：20分

- 媒体类型一致性：10分

- 季集号精准匹配（剧集专属、电影自动满分兜底）：10分

**分项精准计分规则**

- **1\. 标题相似度（0\-60分）**：同时比对本地清洗标题与TMDB中文标题、英文原版标题，取最高相似度得分；完全匹配满分，模糊匹配按字符串相似度比例折算，差异过大/无有效匹配得0分。

- **2\. 年份精度（0\-20分）**：年份完全一致得20分；年份相差1年得15分；相差2年及以上、无年份、年份解析异常，直接0分。

- **3\. 媒体类型一致性（0\-10分）**：本地解析类型（电影/剧集）与TMDB资源类型一致满分；跨类型匹配直接0分，杜绝电影/剧集错配。

- **4\. 季集匹配（剧集专属 0\-10分）**：季号、集号完全一致满分；任意一项不匹配即为0分；识别为电影文件时，该维度固定满分兜底，不影响总分。

**全局兜底 \& 异常计分规则**

- 无TMDB密钥、密钥非法、接口超时、网络异常、服务限流、无搜索结果：匹配度直接赋值 0%。

- 总分自动区间约束：Math\.max\(0, Math\.min\(100, totalScore\)\)，杜绝负数、超百分值。

- 多条TMDB候选结果**独立计分、互不干扰**，前端默认按分值倒序展示。

**分值\-业务状态全局映射（唯一标准，全系统统一复用）**

- ≥85分：高置信度 → 自动采用TMDB标准化元数据回填预览

- 30分 \~ 84分：中低置信度 → 弹窗展示候选列表，人工择优回填

- 0 \~ 29分：极低置信度 → 降级纯本地解析结果，标记「需检查」

# M5 NAS / SMB / NFS 共享目录支持验收清单

## 0. M5-0 共享协议接口架构

- [x] 后端存在统一共享协议接口，至少包含能力描述、配置校验、连接测试、目录浏览、扫描前校验、重命名前校验、路径规范化。
- [x] 本地路径、Windows UNC / SMB、Linux / NAS / Docker / NFS 已挂载路径均通过协议接口接入。
- [x] 扫描服务不直接写死 SMB / NFS 路径判断，必须通过协议注册表调用。
- [x] 重命名 dry-run 和真实执行前校验不直接写死协议判断，必须通过协议注册表调用。
- [x] 目录浏览和连接测试不直接分散实现，必须通过协议接口调用。
- [x] 协议能力接口可返回每种协议的展示名称、凭据能力、目录浏览能力、扫描能力、重命名能力、是否依赖系统挂载、用户说明。
- [x] WebDAV、FTP / SFTP、S3 在能力接口中作为后续候选协议展示，但不提供可执行文件操作。
- [x] 媒体源列表展示连接协议，不只展示路径字符串。
- [x] 系统设置中包含共享协议配置和协议能力说明。
- [x] 前端媒体源表单根据协议能力动态展示字段和用户提示。

## 1. M5-1 媒体源模型与凭据安全

- [x] 媒体源支持三种路径类型：`local` 本地路径 / `unc` Windows UNC (SMB) / `mounted_nfs` Linux-NAS-Docker-NFS 已挂载路径。
- [x] 存量本地媒体源升级后默认 `path_type = local`，原有扫描、预览、重命名流程不受影响。
- [x] UNC 格式 `\\server\share\media` 可保存。
- [x] NAS / Linux / Docker / NFS 已挂载路径 `/volume1/media`、`/mnt/media`、`/mnt/nfs/media` 可通过 `mounted_nfs` 保存。
- [x] 媒体源附属字段可保存和展示：主机、共享名、用户名、域、端口、路径备注。
- [x] `mounted_nfs` 下 NFS 说明字段可保存和展示：NFS 服务器、NFS 导出目录、NFS 版本、本地挂载点、挂载选项。
- [x] 端口仅允许 `0-65535` 数字，非法数字保存拦截。
- [x] SMB 密码或令牌使用密码输入框。
- [x] SMB 凭据使用加密模块保存，数据库不保存明文。
- [x] SMB 凭据在列表、详情、接口响应和日志中全程脱敏。
- [x] SMB 凭据修改采用覆写模式，不支持读取旧值。
- [x] `local` / `mounted_nfs` 类型不显示用户名、密码、域等 SMB 凭据字段。
- [x] `mounted_nfs` 类型媒体源不存储 `username`、`encrypted_secret` 等敏感凭据字段。
- [x] M5 不存在全局独立凭据管理入口或跨媒体源共享凭据库。
- [x] SMB 账号密码仅存储在媒体源自身加密字段中。
- [x] NFS Kerberos 域认证、keytab、principal、realm 等配置项不出现在页面、接口和数据库字段中。
- [x] 支持按路径类型筛选媒体源列表。
- [x] 停用媒体源无法发起扫描任务，前端置灰，后端拦截。

## 2. M5-2 连接测试与目录浏览

- [x] 媒体源新增和编辑页可触发连接测试。
- [x] 本地路径连接测试检查目录存在性、目录类型、读权限和写权限。
- [x] UNC / SMB 连接测试检查路径格式、共享可达性、凭据可用性、读写权限。
- [x] `mounted_nfs` 连接测试检查服务端已挂载路径存在性、目录类型、读写权限。
- [x] `mounted_nfs` 填写 NFS 信息时，连接测试检查本地挂载点存在性、目录类型、读写权限。
- [x] Linux 环境下 `mounted_nfs` 尽可能识别文件系统类型为 `nfs` 或 `nfs4`。
- [x] 无法确认 NFS 文件系统类型时，提示“无法确认文件系统类型”，但不在路径可访问时强制失败。
- [x] 路径正常可读写返回成功提示。
- [x] 路径不存在、无权限、共享离线、凭据错误时返回中文错误和排查建议。
- [x] 测试失败不会删除已保存媒体源配置。
- [x] 连接测试结果写入操作日志，日志不包含明文凭据。
- [ ] 连接测试默认超时使用共享目录配置，默认 `5` 秒。
- [ ] NFS 操作超时使用 NFS 专项配置，默认 `30` 秒。
- [x] 本地、UNC、NAS 挂载、NFS 目录均可加载子目录。
- [x] 目录浏览只返回文件夹，不展示普通媒体文件。
- [x] 目录浏览返回可读、可写权限标识。
- [x] 目录浏览返回数量遵循共享目录配置，默认上限 `500` 条。
- [x] 无权限目录不会造成页面崩溃，返回清晰中文提示。

## 3. M5-3 扫描、重命名校验与状态扩展

- [x] 扫描启动前自动校验媒体源启用状态。
- [x] 扫描启动前自动校验路径存在和可读。
- [ ] SMB / NAS / NFS 目录可正常识别视频文件。
- [x] 共享路径扫描遵守单批数量、批间隔、最小文件阈值等配置。
- [x] 单文件权限报错仅记录失败明细，后续文件继续处理。
- [x] 部分文件失败但任务整体完成时，任务状态为 `partial_completed`，展示文案为“部分完成”。
- [x] 共享目录整体不可访问时任务状态为 `failed`，保存完整故障原因。
- [x] NFS 挂载断连时任务状态为 `connection_lost`，展示文案为“连接中断”。
- [x] NFS `ESTALE` 文件句柄过期时自动重试一次，仍失败则记录文件失败。
- [x] NFS `ETIMEDOUT` 超时时记录日志，继续处理其他文件。
- [x] NFS `EACCES` 权限不足时返回中文权限提示。
- [x] dry-run 前置校验源文件存在、源目录可读、目标目录可写。
- [x] 真实重命名前二次校验文件存在、无同名冲突、目录可写。
- [x] 共享目录只读时直接拦截重命名操作。
- [x] 网络中途断开不会出现“执行成功”误报。
- [x] 完整复用 M3 冲突检测逻辑，不覆盖已有文件。
- [x] dry-run 和二次确认固定开启，前端不可关闭。
- [x] 旧任务状态仍可正常读取和展示。
- [x] 前端遇到未知任务状态时展示“未知状态”，不报错。
- [x] 后端查询、筛选、排序兼容新增状态。

## 4. M5-4 系统设置非 TMDB 分类

### 4.1 文件扫描配置

- [x] 系统设置存在“文件扫描配置”页签。
- [x] 单批数量默认 `100`，仅允许正整数。
- [x] 批间隔秒默认 `1`，允许 `0`，禁止负数。
- [x] 最小扫描文件阈值默认 `0 MB`，前端 MB 展示，后端字节存储。
- [x] 是否跳过隐藏文件默认开启。
- [x] 是否递归扫描子目录默认开启。
- [x] 扫描前路径校验默认开启。
- [x] 修改配置仅对后续扫描任务生效，运行中任务不受影响。

### 4.2 解析重命名配置

- [x] 系统设置存在“解析重命名配置”页签。
- [x] 电影命名模板默认 `{title}.{year}`。
- [x] 剧集命名模板默认 `{title}.{year}.S{season:02d}E{episode:02d}`。
- [x] 文件分隔符默认 `.`。
- [x] 是否保留年份默认开启。
- [x] 非法字符清理强制开启，不可关闭。
- [x] 列表文本截断字节默认 `50`。
- [x] 路径展示截断字节默认 `80`。
- [x] 模板禁止为空或纯特殊字符，保存时后端拦截。
- [x] 修改模板不会覆盖已手动编辑的命名预览条目。
- [x] 文本和路径截断仅影响列表展示，不影响真实文件名和真实路径。

### 4.3 系统运维配置

- [x] 系统设置存在“系统运维配置”页签。
- [x] 日志保留天数默认 `30`，只允许非负数字。
- [x] 日志默认加载条数默认 `200`，只允许正整数。
- [x] 重命名前强制 dry-run 固定开启，只读展示。
- [x] 重命名前二次确认固定开启，只读展示。
- [x] 失败日志持久化默认开启。
- [x] 批量操作上限默认 `200`，只允许正整数。

### 4.4 共享目录配置

- [x] 系统设置存在“共享目录配置”页签。
- [x] 默认路径类型可下拉选择。
- [x] 全局连接测试超时默认 `5` 秒，数字合法性校验。
- [x] 目录浏览单次上限默认 `500`，数字合法性校验。
- [x] 扫描强制连通校验默认开启。
- [x] 重命名前写权限前置校验默认开启。
- [x] NFS 操作超时默认 `30` 秒。
- [x] NFS 重试次数默认 `3`。
- [x] 启用 NFSv4 优先默认开启。
- [x] NFS 挂载检查间隔默认 `60` 秒。
- [x] 全局共享目录配置不保存任何 SMB 账号密码。

## 6. 边界功能验收

- [x] 媒体源表单按路径类型展示能力说明，用户能区分本地路径、SMB、挂载路径、NFS 的差异。
- [x] SMB 类型明确提示：凭据加密保存，用于连接测试和共享访问校验。
- [x] Linux / NAS / Docker 挂载路径明确提示：程序只访问服务端或容器内路径，宿主机到容器映射需用户提前配置。
- [x] `mounted_nfs` 类型明确提示：不保存用户名密码，不自动挂载，依赖系统已挂载路径和 UID/GID 权限。
- [x] 选择 `mounted_nfs` 挂载路径类型时，表单隐藏用户名和密码输入项。
- [x] 仅选择 Windows UNC / SMB 类型时，才展示账号和密码输入框。
- [x] `mounted_nfs` 媒体源无敏感凭据存储，日志不会输出密钥类信息。
- [ ] 填入 WebDAV 路径时，前端提示当前版本暂不支持 WebDAV 协议，并说明已列入后续候选能力。
- [ ] 填入 FTP / SFTP 路径时，前端提示当前版本暂不支持，并说明已列入后续候选能力。
- [ ] 填入 S3 路径时，前端提示当前版本暂不支持对象存储路径，并说明已列入后续候选能力。
- [x] 文档包含 WebDAV / FTP / SFTP / S3 后续支持前必须补充的设计项。
- [x] 系统不会自动执行 `mount`、`net use` 等挂载命令。
- [x] Windows 环境选择 NFS 类型时，提示需要先在系统层准备可访问挂载点，或建议使用 SMB。
- [x] 账号密码不会输出到普通运行日志，全程脱敏。
- [x] 不存在跨设备文件迁移功能入口。

## 8. 异常场景综合验收

- [ ] 扫描过程中共享目录突然断开：任务标记失败或连接中断，不丢失已扫描记录。
- [ ] 扫描过程中网络闪断恢复：任务继续完成，失败文件记录明细。
- [ ] 重命名过程中目标文件被其他进程占用：操作失败，记录冲突原因。
- [ ] 重命名过程中源文件被删除：操作失败，记录文件不存在。
- [x] 批量操作超过上限：前端拦截，提示批量操作上限。
- [ ] 并发扫描多个媒体源：各任务独立运行，互不干扰。
- [ ] 系统重启后未完成任务状态正确恢复或明确标记为失败。

# M5 实机验收备忘

## 1. 验收目的

M5 的自动化测试已经覆盖共享协议接口、配置校验、连接测试基础逻辑、扫描前校验、重命名前校验和前端基础交互。以下项目必须在真实部署环境中验证，避免本地测试无法覆盖的网络、权限、挂载和 NAS 行为差异。

## 2. Windows UNC / SMB

- [ ] 在 Windows 后端服务所在主机上配置可访问共享，例如 `\\NAS\Movies`。
- [ ] 使用当前后端服务运行用户访问共享目录，确认可读。
- [ ] 使用当前后端服务运行用户在共享目录中新建和删除临时文件，确认可写。
- [ ] 在媒体源中选择 Windows UNC / SMB，填写 UNC 路径并保存。
- [ ] 点击“测试连接”，成功时返回可读、可写状态。
- [ ] 使用错误 UNC 路径测试，确认返回中文错误和排查建议。
- [ ] 使用无权限共享测试，确认扫描任务无法启动或返回权限错误。
- [ ] 确认日志中不出现 SMB 明文密码。
- [ ] 确认系统不会自动执行 `net use`，共享登录仍由操作系统或服务运行用户负责。

## 3. Linux / NAS 已挂载路径

- [ ] 在 Linux、NAS 或 fnOS 上准备已挂载目录，例如 `/mnt/media`、`/volume1/video`、`/share/Movies`。
- [ ] 确认后端服务运行用户可以读取目录。
- [ ] 确认后端服务运行用户可以写入目录。
- [ ] 在媒体源中选择“已挂载路径 / NFS”，填写本机可访问路径并保存。
- [ ] 点击“测试连接”，成功时返回可读、可写状态。
- [ ] 使用不存在路径测试，确认返回中文错误和修改建议。
- [ ] 使用只读路径测试，确认重命名前置校验会拦截。

## 4. NFS 挂载

- [ ] 在 Ubuntu / Debian 环境安装 `nfs-common` 并完成 NFS 挂载。
- [ ] 在 CentOS / RHEL 环境安装 `nfs-utils` 并完成 NFS 挂载。
- [ ] 在 Alpine 环境安装 `nfs-utils` 并完成 NFS 挂载。
- [ ] 验证临时挂载命令可用。
- [ ] 验证 `/etc/fstab` 永久挂载配置可用。
- [ ] 在媒体源中选择“已挂载路径 / NFS”，填写挂载点、NFS 服务器和导出目录。
- [ ] 点击“测试连接”，确认路径可访问。
- [ ] Linux 环境下确认可识别 `nfs` 或 `nfs4` 文件系统类型；无法确认时应只提示，不强制失败。
- [ ] 模拟 NFS 断开，确认扫描任务标记为失败或连接中断，不误报成功。
- [ ] 模拟 `ESTALE` 文件句柄过期，确认失败原因可读。
- [ ] 确认 NFS 类型不展示用户名和密码输入框，不保存敏感凭据。
- [ ] 确认系统不会自动执行 `mount`。

## 5. Docker / NAS 容器部署

- [ ] 宿主机先挂载 NFS 或 NAS 共享路径。
- [ ] 使用 Docker bind mount 将宿主机路径映射到容器内路径。
- [ ] 媒体源填写容器内路径，而不是宿主机路径。
- [ ] 检查容器运行用户 UID / GID 与共享目录权限匹配。
- [ ] 测试容器内路径可读、可写。
- [ ] 发起扫描任务，确认能识别视频文件。
- [ ] 执行 dry-run，确认目标目录写权限校验正常。
- [ ] 执行单条安全重命名，确认真实文件被正确重命名且不覆盖已有文件。

## 6. 业务回归

- [ ] M1 媒体源、扫描任务、扫描结果正常。
- [ ] M2 命名预览生成、查询、编辑正常。
- [ ] M3 多选、全部和单条安全重命名正常。
- [ ] M4 TMDB 匹配和待处理文件列表正常。
- [ ] M5 本地路径、SMB、NFS 三类媒体源切换时页面无明显错位。
- [ ] 停用媒体源无法发起扫描，前端置灰，后端拦截。

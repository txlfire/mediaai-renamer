# M5 NAS / SMB / NFS 共享目录支持开发计划

## 1. 阶段目标

M5 在 M4 已有本地目录扫描、命名预览、TMDB 匹配和安全重命名基础上，补齐 NAS / SMB / NFS 共享目录接入能力。

目标是让部署在 Windows、Linux、Docker、NAS 或 fnOS 环境中的服务，可以稳定扫描和操作共享目录，并在真实文件操作前完成路径、权限、连接状态和凭据安全校验。

M5 同时补齐系统设置菜单中除 TMDB 以外的配置分类，避免系统设置长期只有一个真实可用页签。

## 2. 设计原则

- 共享目录先作为服务端可访问路径处理，不在前端直接访问共享文件。
- 真实文件操作必须继续走 M3 的 dry-run 和用户确认流程。
- 程序不自动执行系统级 `mount`、`net use` 等挂载命令，只校验当前服务进程是否可访问路径。
- SMB 凭据仅用于连接测试和后续共享访问校验，复用媒体源自身加密字段保存，必须加密保存、脱敏展示、禁止写入普通日志。
- NFS 依赖操作系统挂载和 UID/GID 权限模型，不显示用户名、密码输入框，不保存密码类凭据。
- M5 不开发全局独立凭据管理模块，不建立跨媒体源共享的凭据库。
- M5 暂不支持 NFS Kerberos 域认证相关配置与密钥存储。
- 网络异常、权限不足、共享断开时返回可读中文提示，不导致任务崩溃。
- Docker / NAS 场景必须明确宿主机路径和容器路径的映射关系。

## 3. 本期范围

### M5-0 共享协议接口架构

M5 开发时必须先抽象共享访问协议接口，再接入具体协议实现。扫描、目录浏览、连接测试、dry-run 和真实重命名不得直接写死 SMB / NFS / 本地路径判断。

建议后端接口结构：

```text
backend/app/service/shared_protocols/
  base.py              # 协议接口、能力描述、测试结果模型
  local.py             # 本地路径和普通挂载路径实现
  smb.py               # SMB / UNC 实现
  mounted_nfs.py       # Linux / NAS / Docker / NFS 已挂载路径实现
  registry.py          # 根据 path_type 获取协议实现
  errors.py            # 协议错误和中文提示映射
```

协议接口至少包含以下能力：

| 方法 | 用途 | 说明 |
| --- | --- | --- |
| `capabilities()` | 返回协议能力 | 告诉前端和业务层是否支持扫描、重命名、目录浏览、凭据、挂载检测 |
| `validate_config()` | 校验媒体源配置 | 校验路径类型、必填字段、端口、凭据状态和 NFS 字段 |
| `test_connection()` | 连接测试 | 返回成功/失败、中文提示、排查建议 |
| `list_directories()` | 目录浏览 | 只返回目录和权限状态 |
| `check_scan_ready()` | 扫描前校验 | 校验存在、可读、连接状态 |
| `check_rename_ready()` | 重命名前校验 | 校验源可读、目标目录可写、共享未断开 |
| `normalize_path()` | 路径规范化 | 统一不同协议的展示路径和实际访问路径 |

协议能力描述必须包含：

- `protocol`：协议标识，如 `local`、`smb`、`mounted_nfs`。
- `display_name`：前端展示名称。
- `supports_credentials`：是否需要在媒体源中配置凭据。
- `supports_directory_browse`：是否支持目录浏览。
- `supports_scan`：是否支持扫描。
- `supports_rename`：是否支持真实重命名。
- `requires_system_mount`：是否依赖系统预先挂载。
- `can_verify_filesystem_type`：是否可校验文件系统类型。
- `future_candidate`：是否属于后续候选协议。
- `user_notice`：给用户看的能力边界说明。

M5 只实现：

- `local`。
- `smb`。
- `mounted_nfs`。

WebDAV、FTP / SFTP、S3 先注册为后续候选协议能力说明，不实现真实连接和文件操作。

### M5-1 共享路径模型和凭据安全

扩展媒体源模型，M5 落地路径类型固定为三类：

- `local`：本机路径。
- `unc`：Windows UNC / SMB 路径，如 `\\server\share\media`。
- `mounted_nfs`：服务端已挂载路径，覆盖 Linux / NAS / Docker bind mount / NFS 本地挂载点，如 `/mnt/media`、`/volume1/media`、`/mnt/nfs/media`。

表单联动规则：

- `path_type = unc`：展示 SMB 主机、共享名、域、用户名、密码等字段；密码加密入库，接口和日志全程脱敏。
- `path_type = local`：隐藏账号、密码、域等凭据输入项，仅保存本地路径和基础备注。
- `path_type = mounted_nfs`：隐藏账号、密码、域等凭据输入项；展示服务端可访问路径，可选记录 NFS 服务器 IP、NFS 导出目录、版本、挂载选项等说明字段。

媒体源增加共享路径字段：

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `path_type` | enum | `local` / `unc` / `mounted_nfs` |
| `host` | string | SMB 主机或 NAS 主机，可选 |
| `share_name` | string | SMB 共享名，可选 |
| `domain` | string | SMB 域，可选 |
| `username` | string | SMB 用户名，可选 |
| `encrypted_secret` | string | SMB 密码或令牌密文，不回显 |
| `port` | integer | 端口，范围 `0-65535` |
| `remark` | string | 路径备注 |
| `nfs_host` | string | `mounted_nfs` 下可选，NFS 服务器 IP 或域名 |
| `nfs_export` | string | `mounted_nfs` 下可选，NFS 导出目录，如 `/volume1/media` |
| `nfs_version` | enum | `mounted_nfs` 下可选，`v3` / `v4` / `v4.1`，默认 `v4` |
| `nfs_options` | string | `mounted_nfs` 下可选，NFS 挂载选项，仅记录展示，不用于自动挂载 |
| `local_mount_path` | string | `mounted_nfs` 下可选，NFS 本地挂载点；未填写时以 `path` 作为实际访问路径 |

SMB 凭据安全要求：

- SMB 账号密码存储复用媒体源自身加密字段，不新增全局独立凭据管理模块。
- 媒体源加密字段使用部署环境密钥进行对称加密。
- 环境变量未配置密钥时，开发环境可使用本地默认派生密钥，但生产部署文档必须要求显式配置。
- 数据库只保存密文和是否已配置凭据的状态。
- API 返回时不返回明文，只返回脱敏状态。
- 修改凭据采用覆写模式，不支持读取旧值。
- 日志、异常、工作日志和提交记录不得包含明文密码。

`mounted_nfs` / NFS 凭据边界：

- `mounted_nfs` 类型媒体源不提供用户名、密码、域、令牌等输入框。
- 仅选择 Windows UNC / SMB 类型时展示账号、密码输入框；本地、`mounted_nfs` 类型均不展示账号、密码输入框。
- `mounted_nfs` 类型媒体源不写入 `username`、`encrypted_secret` 等敏感凭据字段。
- NFS 挂载认证由操作系统和 NAS 侧配置负责，系统只校验本地挂载点可访问性。
- M5 不支持 Kerberos NFS，不保存 keytab、principal、realm 等 Kerberos 配置或密钥。

数据库迁移要求：

- 新增字段必须通过 schema migration 升级。
- 存量媒体源默认迁移为 `path_type = local`。
- 旧数据中的 `path`、`name`、`enabled` 必须保持兼容。
- 迁移后旧媒体源仍可扫描、预览和重命名。

### M5-2 连接测试和目录浏览

新增共享目录连接测试接口。

连接测试按路径类型处理：

- 本地路径：检查路径存在、目录类型、可读、可写。
- UNC / SMB：检查路径格式、共享可达、凭据可用、目录读写权限。
- mounted_nfs：检查服务端已挂载路径存在、目录类型、可读、可写；填写 NFS 信息时尽可能确认文件系统类型。

目录浏览能力：

- 本地、UNC、mounted_nfs 都使用统一目录浏览接口。
- 只返回目录，不返回普通文件。
- 每次返回数量受共享目录配置上限控制，默认最多 `500` 条。
- 返回目录可读、可写状态。
- 无权限、离线、路径不存在时返回中文错误和排查建议。

连接测试超时口径：

- 全局共享目录连接测试默认超时：`5` 秒。
- 本地路径、UNC / SMB、NFS 可在配置中设置类型级覆盖值。
- NFS 文件操作超时默认 `30` 秒，作为 NFS 专项配置，不与普通连接测试超时混用。

### M5-3 NFS 检测策略

NFS 不像 SMB 一样通过用户名密码登录。程序实际能做的是确认“服务进程当前访问的这个本地路径是否像一个正常挂载点，并且可读写”。

检测分三层：

1. 基础路径检测：所有系统都执行，检查本地挂载点是否存在、是否目录、是否可读写。
2. Linux 文件系统类型检测：优先读取 `/proc/mounts` 或等价系统信息，确认挂载点类型是否为 `nfs` 或 `nfs4`。
3. 降级检测：Windows 或无法读取系统挂载信息时，不判定为失败，只提示“无法确认文件系统类型”，继续以路径可访问性作为判断依据。

不采用 `mount -l` 作为首选实现，原因：

- 需要额外命令权限和不同发行版兼容处理。
- 命令输出格式不稳定。
- 不能在容器内可靠反映宿主机挂载状态。

M5 不执行自动挂载。用户需要先在宿主机或系统中完成 NFS 挂载，再把本地挂载点配置到媒体源中。

### M5-4 网络协议能力边界

不同网络协议的能力不同，系统必须按能力区分，不把所有网络路径都当成同一种目录处理。

| 协议 / 类型 | M5 支持状态 | 路径形态 | 凭据方式 | 扫描 | 重命名 | 目录浏览 | 用户提示重点 |
| --- | --- | --- | --- | --- | --- | --- | --- |
| 本地路径 | 支持 | `D:\Media`、`/data/media` | 无 | 支持 | 支持 | 支持 | 服务所在机器必须能访问该路径 |
| SMB / Windows UNC | 支持 | `\\server\share\media` | 保存加密凭据用于连接测试 | 支持 | 支持 | 支持 | 密码加密保存；网络断开或权限不足会阻止扫描和重命名 |
| Linux / NAS / Docker / NFS 已挂载路径 | 支持，`path_type = mounted_nfs` | `/mnt/media`、`/volume1/media`、`/mnt/nfs/media`、容器内 bind mount | 由操作系统、宿主机或 NAS 管理，不保存凭据 | 支持 | 支持 | 支持 | 程序只看到本地挂载点；NFS 需先完成系统挂载；容器内外路径必须映射正确 |
| WebDAV | M5 不实现，后续候选 | `https://host/dav/path` | 用户名密码、Token 或服务端配置 | 后续评估 | 后续评估，需重点处理远程写入和锁 | 后续评估 | 不是本地文件系统，不能直接复用本地 `rename` 语义 |
| FTP / SFTP | M5 不实现，后续候选 | `ftp://...`、`sftp://...` | 用户名密码、密钥 | 后续评估 | 后续评估，需处理远程移动、覆盖和断点失败 | 后续评估 | 远程文件操作语义与本地文件不同，失败恢复更复杂 |
| S3 / 对象存储 | M5 不实现，后续候选 | `s3://bucket/prefix` | Access Key / Secret Key / IAM | 后续评估 | 后续评估，通常是复制加删除，不是真正重命名 | 后续评估 | 对象存储没有真实目录和原子重命名，需要单独设计 |

用户告知要求：

- 媒体源表单必须显示当前路径类型的能力说明。
- 用户输入 WebDAV / FTP / SFTP / S3 路径时，明确提示“当前版本暂不支持该协议，已列入后续候选能力”。
- 对支持的共享路径，提示“程序不会自动挂载，请确认服务所在机器或容器已经能访问该路径”。
- 对真实文件操作，统一提示“重命名前会执行 dry-run 和权限校验，不会覆盖已有文件”。

### M5-5 扫描、dry-run 和重命名前校验

扫描任务启动前：

- 校验媒体源启用状态。
- 校验路径存在和可读。
- 对 UNC / SMB 和 NFS 执行连接状态检查。
- 校验失败时不启动扫描，任务状态记录失败原因。

扫描过程中：

- 单文件权限错误记录失败明细，继续处理后续文件。
- 共享目录整体不可访问时任务失败。
- 网络闪断后恢复时，可继续处理的文件继续处理，不可处理的文件记录失败。

NFS 异常处理：

- `ESTALE`：文件句柄过期，自动重试一次，仍失败则记录文件失败。
- `ETIMEDOUT`：操作超时，记录错误，继续处理其他文件。
- `EACCES`：权限不足，按文件失败或任务失败处理，取决于发生位置。

重命名前：

- dry-run 前校验源文件存在、源目录可读、目标目录可写。
- 真实执行前二次校验源文件、目标文件和目录写权限。
- 共享目录只读时禁止执行真实重命名。
- 网络断开时返回失败，不误报成功。
- 完整复用 M3 冲突检测，不覆盖已有文件。

### M5-6 任务状态扩展

M5 新增或明确以下任务状态：

| 状态值 | 展示文案 | 适用场景 |
| --- | --- | --- |
| `pending` | 等待中 | 任务已创建但未开始 |
| `running` | 扫描中 | 正在扫描 |
| `completed` | 已完成 | 全部处理成功 |
| `partial_completed` | 部分完成 | 部分文件失败，但任务整体完成 |
| `failed` | 失败 | 任务无法继续 |
| `connection_lost` | 连接中断 | 共享目录或 NFS 挂载中途不可访问 |

兼容要求：

- 旧数据库中的既有状态值保持原样可展示。
- 前端未知状态统一展示为“未知状态”，不得空白或报错。
- 后端查询、筛选、排序必须兼容新增状态。
- 迁移脚本不得强制改写旧任务状态。

测试要求：

- 覆盖旧状态读取。
- 覆盖新增状态展示。
- 覆盖共享断开时任务状态写入。
- 覆盖“部分完成”统计和错误信息保存。

### M5-7 系统设置非 TMDB 分类

#### 5.6.1 文件扫描配置

- 单批数量：默认 `100`，必须为正整数。
- 批间隔秒数：默认 `1`，允许 `0`，不得为负数。
- 最小扫描文件阈值：默认 `0 MB`，前端按 MB 展示，后端按字节保存。
- 是否跳过隐藏文件：默认开启。
- 是否递归扫描子目录：默认开启。
- 扫描前路径可用性校验：默认开启，共享目录强制开启。

#### 5.6.2 解析重命名配置

- 电影命名模板：默认 `{title}.{year}`。
- 剧集命名模板：默认 `{title}.{year}.S{season:02d}E{episode:02d}`。
- 文件名分隔符：默认 `.`。
- 是否保留年份：默认开启。
- 非法字符清理：强制开启，不允许关闭真实重命名前的安全清理。
- 文本截断显示字节数：默认 `50`。
- 路径展示字节数：默认 `80`。

M5 只完成基础表单、校验和热配置结构；复杂可视化命名规则后移到 M8。

#### 5.6.3 系统运维配置

- 日志保留天数：默认 `30` 天。
- 操作日志页面默认加载条数：默认 `200`。
- 重命名前强制 dry-run：固定开启，只读展示。
- 执行前二次确认：固定开启，只读展示。
- 失败任务保留详细错误：默认开启。
- 批量操作上限：默认 `200`。

#### 5.6.4 共享目录配置

- 默认路径类型。
- 全局连接测试超时：默认 `5` 秒。
- 目录浏览最大返回数量：默认 `500`。
- 扫描前强制连接测试：默认开启。
- 重命名前强制写权限测试：默认开启。
- NFS 操作超时：默认 `30` 秒。
- NFS 重试次数：默认 `3`。
- 启用 NFSv4 优先：默认开启。
- NFS 挂载检查间隔：默认 `60` 秒。

全局配置不保存任何 SMB 账号密码。

### M5-8 Docker / NAS / NFS 部署文档

文档必须补充：

- Windows UNC 路径示例。
- Linux mount 路径示例。
- 群晖、威联通、fnOS 路径示例。
- Docker 宿主机路径和容器路径映射说明。
- 容器 UID/GID 权限匹配方案。
- NFS 客户端安装指南。
- NFS 临时挂载和 `/etc/fstab` 永久挂载示例。
- Docker 使用 NFS 的推荐方式：宿主机先挂载，再通过 bind mount 映射进容器。
- 路径不可见、权限不足、共享断开、NFS 文件句柄过期等故障排查。

## 4. 明确不做

- 不自动执行系统级 SMB / NFS 挂载命令。
- M5 不实现 WebDAV、FTP、SFTP、S3 协议接入；这些协议保留为后续候选能力。
- 不做跨设备文件迁移功能入口。
- 不绕过 M3 dry-run 和用户确认流程。
- 不在全局配置中保存 SMB 密码。
- 不开发全局独立凭据管理模块；SMB 凭据只保存在对应媒体源自身加密字段中。
- 不支持 NFS Kerberos 域认证、keytab、principal、realm 等配置和密钥存储。
- 不在 M5 实现登录、权限审批、回滚和审计。

## 5. 建议拆分版本

- M5-2：连接测试、目录浏览和路径权限状态。

| 风险 | 处理方式 |
| --- | --- |
| SMB 凭据泄露 | 复用媒体源加密字段保存、脱敏返回、日志过滤、覆写式修改，不建立全局凭据库 |

| 旧数据库升级风险 | schema migration 和旧媒体源兼容测试 |
| 状态枚举扩展影响旧页面 | 前端未知状态兜底展示，后端保留旧值 |

### 独立凭据管理模块评估

M5 结论：暂不建设全局独立凭据管理模块。

理由：

- M5 只有 SMB 需要应用层账号密码，NFS / mounted_nfs 不需要应用保存密码，单独建凭据库收益不高。
- 当前没有用户权限体系和审计体系，独立凭据管理会引入“谁能查看、修改、复用凭据”的新权限问题。
- 全局凭据库会增加迁移、密钥轮换、脱敏、误删除影响范围、跨媒体源引用一致性等复杂度，超出 M5 主要目标。
- 现阶段用媒体源自身加密字段即可满足连接测试和共享访问校验，风险面更小。

后续触发条件：

- WebDAV / FTP / SFTP / S3 开始正式接入，并出现多媒体源复用同一账号、Token 刷新、SSH Key、Access Key / Secret Key 等多种凭据类型。
- 系统具备用户权限、操作审计和凭据变更记录。
- 需要凭据轮换、到期提醒、连接池复用或集中吊销。

预估实现成本：

- 轻量版：新增 `credentials` 表、凭据加密服务、引用关系、脱敏接口、基础测试，约 1-2 天。
- 完整版：增加权限控制、审计日志、密钥轮换、凭据复用管理、前端管理页、迁移和回归测试，约 4-7 天。

建议：M5 只把协议接口和媒体源字段设计成可平滑升级，不提前实现独立凭据管理模块。

# M9 权限、历史、回滚、审计和任务治理验收清单

## 0. 验收口径

- M9 只验收已经落地的身份、权限、审计、回滚和任务治理能力。
- 用户手册不得写入未实现的企业级认证、远程身份协议或多租户能力。
- 所有真实文件操作必须继续经过 dry-run 和用户确认。

## 2. 认证和用户

- [x] 无用户旧库升级后，可创建首个本地管理员。

- [x] 密码只保存哈希，不保存明文。
- [x] 登录失败返回明确中文提示。
- [x] 禁用用户无法登录。

## 3. 权限守卫

- [x] 初始化用户后，未登录用户不能保存系统设置。
- [x] 初始化用户后，缺少 `settings:write` 的用户不能保存系统设置。
- [x] 初始化用户后，缺少 `source:write` 的用户不能新增、编辑、删除媒体源。
- [x] 初始化用户后，缺少 `scan:run` 的用户不能创建扫描任务。
- [x] 初始化用户后，未登录用户不能执行真实重命名。
- [x] 初始化用户后，缺少 `rename:execute` 的用户不能执行真实重命名。
- [x] 未登录用户不能执行回滚。
- [x] 已接入守卫的敏感接口在权限不足时返回 403。
- [x] 初始化用户后，缺少 `metadata:submit` 的用户不能发起 TMDB / AI 外部提交。
- [x] 初始化用户后，缺少 `metadata:submit` 的用户不能覆盖外部提交拦截记录。
- [x] 前端按权限禁用已接入守卫的不可执行按钮，并展示原因。

- [x] 登录成功写入审计事件。
- [x] 登录失败写入审计事件。

- [x] 媒体源新增、编辑、删除写入审计事件。

## 5. 操作运行日志

- [x] 新库会创建 `operation_logs` 表。
- [x] 操作日志查询接口支持按 `task_type`、`task_id` 查询。
- [x] 操作日志查询接口支持 `after_id` 增量拉取。
- [x] 操作日志查询接口默认单次返回不超过 100 条，最大不超过 500 条。
- [x] 操作日志清理跟随系统日志保留天数设置。
- [x] 操作日志清理支持总量上限，默认不超过 128 MB。
- [x] 操作日志清理支持单任务上限，默认不超过 16 MB。
- [x] 操作日志清理支持单次批量上限，默认每批不超过 1000 条。
- [x] 操作日志写入记录 `approx_bytes` 或等价估算大小，用于按容量清理。
- [x] 操作日志清理只删除日志记录，不删除任务、统计和审计事件。
- [x] 超过容量上限时优先清理最早日志，并尽量保留最终状态、error 和 warning。
- [x] 详细日志被部分清理时，摘要接口返回中文提示。
- [x] 日志已清理时，摘要接口返回 `logAvailable=false`、`cleared=true` 和中文提示。
- [x] 操作日志导出接口可导出单个任务的文本日志。
- [x] 扫描任务创建后写入开始日志。
- [x] 扫描任务运行中写入阶段、批量进度和警告日志。
- [x] 扫描普通成功文件不会逐条写入日志，避免日志量失控。
- [x] 扫描任务完成、部分完成或失败时写入最终状态日志。
- [x] 重命名 dry-run 写入校验过程、冲突原因和汇总日志。
- [x] 真实重命名执行写入执行开始、失败项、汇总和最终状态日志。
- [x] 操作日志详情不包含 API Key、token、密码和 SMB 凭据明文。
- [x] 前端触发扫描后自动打开扫描日志窗口。
- [x] 前端触发 dry-run 或真实重命名后自动打开重命名日志窗口。
- [x] 日志窗口支持自动刷新、手动刷新、级别筛选、复制或导出。
- [x] 日志窗口默认轮询间隔不低于 1500 ms，请求失败或页面不可见时降频或停止。
- [x] 日志窗口最多渲染最近 1000 条日志，较早日志折叠并提示隐藏数量。
- [x] 日志已清理时，“查看日志”按钮置灰并显示原因，或弹窗明确显示已清理提示且不继续轮询。
- [x] 任务结束后停止轮询，并允许从扫描任务列表或重命名批次回看日志。

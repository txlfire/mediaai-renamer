# M9 权限、历史、回滚、审计和任务治理验收清单

## 0. 验收口径

- M9 只验收已经落地的身份、权限、审计、回滚和任务治理能力。
- 用户手册不得写入未实现的企业级认证、远程身份协议或多租户能力。
- 所有真实文件操作必须继续经过 dry-run 和用户确认。

## 1. 数据迁移

- [x] 新库会创建 `users` 表。
- [x] 新库会创建 `user_sessions` 表。
- [x] 新库会创建 `audit_events` 表。
- [x] 新库会创建 `rename_rollback_plans` 表。
- [x] 新库会创建 `rename_rollback_items` 表。
- [x] 旧 M8 schema `13` 升级后会补齐 M9 治理表。
- [x] 数据库 schema version 提升到 `14`。

## 2. 认证和用户

- [x] 无用户旧库升级后，可创建首个本地管理员。
- [x] 管理员可登录、退出并查询当前用户。
- [x] 密码只保存哈希，不保存明文。
- [x] 登录失败返回明确中文提示。
- [x] 禁用用户无法登录。
- [x] 会话过期后敏感接口返回 401。
- [x] 前端登录页可完成管理员初始化、登录和退出。
- [x] 前端全局 401 / 403 处理符合中文提示要求。

## 3. 权限守卫

- [x] 初始化用户后，未登录用户不能保存系统设置。
- [x] 初始化用户后，缺少 `settings:write` 的用户不能保存系统设置。
- [x] 初始化用户后，缺少 `source:write` 的用户不能新增、编辑、删除媒体源。
- [x] 初始化用户后，缺少 `scan:run` 的用户不能创建扫描任务。
- [x] 初始化用户后，未登录用户不能执行真实重命名。
- [x] 初始化用户后，缺少 `rename:execute` 的用户不能执行真实重命名。
- [ ] 未登录用户不能执行回滚。
- [x] 已接入守卫的敏感接口在权限不足时返回 403。
- [x] 初始化用户后，缺少 `metadata:submit` 的用户不能发起 TMDB / AI 外部提交。
- [x] 初始化用户后，缺少 `metadata:submit` 的用户不能覆盖外部提交拦截记录。
- [x] 前端按权限禁用已接入守卫的不可执行按钮，并展示原因。

## 4. 审计日志

- [ ] 登录成功写入审计事件。
- [ ] 登录失败写入审计事件。
- [ ] 系统设置保存写入审计事件。
- [ ] 媒体源新增、编辑、删除写入审计事件。
- [ ] 外部提交覆盖保护写入审计事件。
- [ ] 真实重命名执行写入审计事件。
- [ ] 回滚计划创建、dry-run 和执行写入审计事件。
- [ ] 审计详情不包含 API Key、token、密码和 SMB 凭据明文。
- [ ] 审计列表支持按类型、结果、操作者和时间筛选。

## 5. 重命名历史和回滚

- [ ] 只能基于成功的重命名明细创建回滚计划。
- [ ] 回滚计划能展示原路径、当前路径和目标回滚路径。
- [ ] 回滚 dry-run 检查当前文件是否存在。
- [ ] 回滚 dry-run 检查目标路径是否已存在。
- [ ] 回滚 dry-run 检查目标目录写权限。
- [ ] 回滚执行不会覆盖已有文件。
- [ ] 回滚部分失败时，成功和失败条目分别记录结果。
- [ ] 回滚执行后写入审计事件。
- [ ] 回滚失败原因使用中文展示。

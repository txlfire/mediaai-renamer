# M9 权限、历史、回滚、审计和任务治理开发计划

## 1. 阶段目标

M9 聚焦治理能力，在不改变现有扫描、预览、匹配和重命名主流程的前提下，补齐身份、权限、审计、回滚和任务治理。

目标：

- 让高风险操作具备操作者身份和权限判断。
- 让配置变更、外部提交覆盖、真实文件操作和回滚可审计。
- 让已完成的重命名批次具备可验证的反向回滚能力。
- 让用户能集中查看扫描、重命名、回滚和批量操作状态。

## 2. 当前基线

截至 2026-07-09：

- M1-M8 已形成扫描、命名预览、TMDB / AI 回填、安全重命名、增量扫描和命名规则高级配置闭环。
- M1 页面已有当前用户和退出入口占位，但没有真实登录、会话和权限守卫。
- M3 已有重命名批次和明细记录，可作为回滚计划的数据来源。
- M6 外部提交拦截记录已有用户决策和覆盖原因字段，可在 M9 接入真实操作者。
- M7 扫描索引和任务统计已具备任务治理的数据基础。
- M8 模板版本快照已降低命名规则变化对历史预览的追踪风险。

## 3. 拆分建议

### M9-0 设计、迁移和兼容策略

- 新增用户、会话、审计事件、回滚计划和回滚明细表设计。（已完成）
- 明确开发环境和生产环境认证开关策略。
- 明确旧接口兼容策略：只对敏感写操作强制登录，健康检查和静态资源不受影响。
- 增加数据库迁移测试，旧库升级后默认无用户，允许 bootstrap 管理员。（治理表迁移测试已完成，bootstrap 行为待 M9-1 实现）

### M9-1 认证和权限基础

- 实现本地管理员 bootstrap。（后端已完成）
- 实现登录、退出、当前用户接口。（后端已完成）
- 使用安全密码哈希，数据库只保存哈希。（已完成）
- 实现用户权限枚举，权限直接挂在用户记录上，不引入角色模型。（已完成）
- 前端增加登录页、当前用户展示和 401/403 统一处理。（已完成）

### M9-2 敏感接口权限守卫

- 系统设置保存、AI Provider、敏感词、命名规则保存要求 `settings:write`。
- 媒体源新增、编辑、删除要求 `source:write`。
- 扫描任务创建要求 `scan:run`。
- TMDB、IMDb、AI 外部提交要求 `metadata:submit`。
- 真实重命名执行要求 `rename:execute`。
- 前端按权限禁用对应按钮，并展示中文原因。（首批已完成）

### M9-3 审计日志

- 新增审计事件写入服务。
- 登录成功、登录失败、设置保存、媒体源变更、外部提交覆盖、真实重命名执行、回滚操作写入审计。
- 审计详情必须脱敏 API Key、token、密码和 SMB 凭据。
- 前端新增审计日志页面，支持筛选、分页和详情查看。

### M9-4 重命名历史和回滚

- 基于已成功的 `rename_operation_items` 创建回滚计划。
- 回滚 dry-run 检查当前文件、目标路径、冲突和目录写权限。
- 回滚执行复用 M3 安全重命名校验和 M5 共享路径校验。
- 回滚结果写入回滚明细和审计事件。
- 前端提供回滚计划详情、dry-run 结果、执行确认和失败原因展示。

### M9-5 任务治理

- 新增任务治理聚合接口，统一查询扫描任务、重命名批次、回滚计划和批量匹配摘要。
- 前端新增任务治理页面，支持类型、状态、媒体源和时间筛选。
- 支持从任务治理跳转到扫描结果、命名预览、重命名批次或回滚计划详情。
- 对可归档任务提供归档入口；取消和重试只在后端已支持的任务类型开放。

### M9-6 文档和验收

- 更新 README 当前阶段和文档索引。
- 输出 M9 设计手册、验收清单和用户手册。
- 更新完整用户手册，只记录已落地的登录、审计、回滚和任务治理能力。
- 执行后端测试、前端测试、前端构建、编码检查和 `git diff --check`。

## 4. 重点设计约束

- 不绕过 M3 dry-run 和 M5 共享路径校验。
- 不让回滚覆盖已有文件。
- 不把 API Key、token、密码、SMB 凭据写入审计明细。
- 不在 M9 引入企业级多租户和外部身份协议。
- 不让只读用户触发外部提交、真实重命名、回滚和配置保存。
- 开发免登录必须由明确配置控制，不能成为生产默认行为。

## 5. 不做范围

- OAuth、LDAP、OIDC、双因素认证。
- 多租户和媒体源级数据隔离。
- 跨设备会话管理。
- 审计日志远程推送。
- 对外部程序移动、覆盖或删除后的文件做强制恢复。
- WebSocket 实时任务进度。

# M11 远程协议扩展开发计划

## 1. 阶段目标

M11 在 M5 本地、SMB 和已挂载 NFS 媒体源基础上，引入受控的远程协议访问能力。目标不是把远程资源伪装成本地目录，而是按协议真实能力完成连接测试、目录浏览、扫描、命名预览、安全重命名、冲突控制和失败恢复。

首批实际交付顺序为：协议能力模型与凭据迁移、WebDAV、SFTP、S3；FTPS / FTP 仅在安全评估通过后进入兼容范围。

## 2. 当前基线

- M5 已支持 `local`、`unc` 和 `mounted_nfs` 媒体源，SMB 凭据保存在媒体源记录中并执行脱敏处理。
- 扫描、命名预览、dry-run、安全重命名、审计、操作日志、回滚和任务治理已形成闭环。
- M10 已正式发布 `v0.10.7`，多源元数据不改变远程文件操作边界。
- 现有凭据加密为历史兼容实现；远程协议接入前必须升级为版本化密文格式并提供旧密文迁移。

## 3. 范围与边界

### 3.1 纳入 M11

- 统一远程媒体源协议能力模型、凭据模型、连接测试和目录浏览。
- WebDAV 的 HTTPS 扫描、预览、dry-run 和安全重命名。
- SFTP 的密钥或密码认证、扫描、预览、dry-run 和安全重命名。
- S3 兼容对象存储的扫描、预览、dry-run 和以“复制后校验再删除”实现的受控对象键迁移。
- 远程操作锁、租约、幂等操作记录、失败恢复和可审计的过程日志。
- FTPS / FTP 兼容性评估；纯 FTP 默认关闭，不作为默认推荐协议。

### 3.2 不纳入 M11

- 不支持未加密 HTTP WebDAV、明文 FTP 默认连接或绕过证书校验。
- 不把远程目录自动挂载为本地目录，也不调用操作系统 `mount`、`net use`。
- 不实现跨站点复制、媒体库入库、第三方账号市场和多租户凭据托管。
- 不绕过敏感词保护、权限守卫、dry-run、二次确认和审计。

## 4. 核心设计

### 4.1 协议能力模型

每个远程 Provider 必须声明以下能力，业务流程根据能力决定可展示的操作，不能再假设所有路径支持本地 `rename`：

- `browse`：目录浏览。
- `scan`：递归列举媒体文件。
- `read_metadata`：读取大小、修改时间、ETag 或版本号。
- `atomic_rename`：服务端原子重命名。
- `copy_delete_rename`：复制、校验、删除组成的迁移式重命名。
- `conditional_write`：基于 ETag、版本号或修改时间避免覆盖并发修改。
- `resume`：中断后可根据操作记录继续或回滚。

统一接口保持在服务层，扫描和重命名服务只依赖能力接口，不直接拼接协议 URL 或调用第三方 SDK。

### 4.2 凭据与安全

- 媒体源保存协议、端点、根路径、认证方式、加密凭据和密文版本。
- 凭据仅由后端在连接测试、扫描或重命名时解密，不通过 API、审计或运行日志回显。
- 新密文采用带版本号、随机 nonce 和完整性校验的加密格式；提供旧 SMB 密文一次性迁移和失败回退。
- HTTPS / TLS 证书校验默认开启；自签名证书只能由管理员显式信任指纹，不能用全局跳过校验替代。
- S3 密钥、SFTP 私钥和 WebDAV 密码必须按最小权限范围配置。

### 4.3 远程重命名语义

- 所有远程真实写入前必须执行 dry-run，确认源对象版本、目标不存在、父目录可写、协议能力满足。
- WebDAV / SFTP 优先使用服务端原子移动；S3 使用“复制到临时键 → 校验 → 提交目标键 → 删除源键”的可恢复流程。
- 每个媒体源同一时间只允许一个远程写操作；锁采用数据库租约并记录心跳、操作者、任务和过期时间。
- 每个远程条目写入不可变操作标识、源版本和目标版本；重试时先检查已有结果，禁止重复删除或覆盖。
- 失败项保留在任务日志和回滚/恢复入口，不能静默标记成功。

## 5. 阶段拆分

### M11-0 架构、迁移和安全基线

- 设计 `RemoteMediaProvider` 统一接口与协议能力枚举。
- 扩展媒体源表，增加协议端点、认证方式、密文版本、根路径和能力快照。
- 新增远程操作锁和远程操作明细表，支持租约、幂等键、状态、错误与恢复信息。
- 实现版本化凭据加密、旧 SMB 密文迁移、脱敏与迁移回归测试。
- 前端媒体源表单按协议动态展示字段，未支持能力不展示可执行按钮。

验收：旧本地、SMB、已挂载 NFS 媒体源可无损迁移；凭据不明文返回；并发写操作被明确拒绝并保留审计记录。

### M11-1 WebDAV

- 支持 HTTPS WebDAV Basic / Digest 或 Bearer 认证、连接测试和目录浏览。
- 支持 PROPFIND 递归扫描，读取大小、修改时间、ETag 并进入现有扫描索引。
- 支持 MOVE dry-run 与真实重命名，使用 `If-Match` 或等价版本条件防止覆盖。
- 接入任务日志、重命名批次、回滚计划和权限守卫。

验收：WebDAV 媒体源可在真实或容器化服务中完成连接、扫描、预览、dry-run、重命名和失败恢复演练。

### M11-2 SFTP

- 支持主机密钥校验、密码或私钥认证、连接测试和目录浏览。
- 支持递归扫描和原子 `rename`；不支持原子操作的服务器必须降级为明确拒绝，不隐式复制删除。
- 支持远程权限、断连、路径编码和文件占用错误的中文分类。

验收：SFTP 媒体源完成连接、扫描、预览、dry-run、重命名和主机密钥变更拦截测试。

### M11-3 S3 / 对象存储

- 支持 AWS S3 与明确声明兼容性的 S3 Endpoint、Bucket、Prefix、Region、Access Key / Secret Key 配置。
- 支持 ListObjects 扫描、对象版本 / ETag 读取和目录前缀展示。
- 使用复制、校验、提交、删除的状态机实现对象键迁移，支持中断恢复和幂等重试。
- 处理版本化 Bucket、最终一致性、Multipart ETag 与对象锁限制。

验收：MinIO 集成环境完成扫描和对象键迁移；中断、重复提交、目标冲突和删除失败均可追溯和恢复。

### M11-4 FTPS / FTP 兼容性评估

- 输出协议安全与能力矩阵，优先支持显式 TLS 的 FTPS。
- 仅在管理员确认风险后允许纯 FTP 连接；纯 FTP 默认禁用真实重命名。
- 根据评估结果决定是否在 M11 实装只读扫描，或整体后移到独立兼容阶段。

验收：文档明确支持边界；任何不安全配置都不会在默认路径下发送凭据或执行写操作。

### M11-5 前端、文档和验收

- 媒体源管理增加协议选择、连接状态、能力提示和凭据脱敏展示。
- 扫描任务、命名预览、重命名批次和任务治理展示协议、锁状态、恢复状态与失败原因。
- 新增 M11 设计手册、验收清单、用户手册、部署说明和实机验证记录。
- 执行后端测试、前端测试、协议集成测试、前端构建、编码检查、打包检查和 `git diff --check`。

## 6. 关键风险与控制

| 风险 | 控制措施 |
| --- | --- |
| 远程协议不具备本地原子重命名语义 | Provider 能力声明；不满足安全条件时只允许扫描或明确拒绝。 |
| 网络中断导致重复移动或删除 | 幂等键、操作状态机、源/目标版本校验和恢复入口。 |
| 凭据泄露或被发送到错误主机 | 版本化加密、固定协议校验、主机密钥/证书指纹校验、日志脱敏。 |
| 多任务并发修改同一媒体源 | 数据库租约锁、心跳、超时恢复和审计。 |
| S3 复制删除不等价于重命名 | 显式状态机、复制后校验、目标提交成功后才删除源对象。 |
| FTP 明文传输 | 默认禁用；优先 FTPS，纯 FTP 仅管理员风险确认后的受限兼容。 |

## 7. API 和数据变更方向

计划新增或扩展：

```http
POST /api/media-sources/{id}/test-connection
GET  /api/media-sources/{id}/browse
GET  /api/media-sources/{id}/capabilities
GET  /api/remote-operations/{operation_id}
POST /api/remote-operations/{operation_id}/recover
```

媒体源保存接口保持现有路径，按协议类型增加受控字段；旧字段继续兼容。远程重命名仍复用现有 `dry-run → 确认 → 执行` 流程，不新增绕过安全检查的写接口。

```powershell

npm.cmd run frontend:test
npm.cmd run frontend:build
npm.cmd run check:encoding
git diff --check
docker compose config
docker compose -f docker-compose.ghcr.yml config
```

协议集成测试采用独立容器或可控测试服务：WebDAV、SFTP、MinIO；不得使用真实用户媒体目录或真实生产凭据。

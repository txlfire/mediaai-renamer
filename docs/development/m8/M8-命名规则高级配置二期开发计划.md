# M8 命名规则高级配置二期开发计划

## 1. 阶段目标

M8 聚焦命名规则高级配置二期，在不改变现有重命名主流程的前提下，把命名模板从“可编辑”推进到“可管理、可测试、可版本化”。

目标：

- 在现有 `NamingTemplateBuilder` 基础上补齐模板版本、导入导出、双语命名和规则测试能力。
- 保持现有模板配置和预览接口兼容，避免影响 M1-M7 已落地流程。
- 让用户在真正重生成预览前，就能看见规则变更带来的命名差异。

## 2. 当前实现基线

截至 2026-07-08 开发推进，代码中已经具备以下 M8 能力，当前进入 M8-4 文档与验收收尾：

- 命名模板热配置已包含电影和剧集模板版本号、更新时间字段。
- 后端已存在命名模板导出、导入、规则测试和差异预览接口，规则测试会返回字段命中情况和格式告警。
- 前端系统设置页已接入命名规则测试、差异预览、导入、导出、字段命中、格式告警和版本展示入口。
- M8-3 已实现预览链路模板版本快照，历史预览可判断当前、旧模板或版本未知状态。
- README、总设计文档、用户手册和验收清单随 M8-4 收尾同步。

## 3. 拆分建议

### M8-0 配置模型与兼容迁移

- 梳理当前命名模板热配置结构，补充模板版本号、配置更新时间和操作者信息。
- 为电影、剧集模板补充导入导出 JSON 结构版本字段。
- 保持旧配置可直接读取；旧数据缺少版本字段时自动按 `v1` 兼容。
- 为模板测试、差异预览预留独立接口模型，避免直接污染正式预览表。

### M8-1 后端命名规则服务

- 新增模板导入校验、导出下载和结构校验逻辑。
- 新增规则测试接口，输入样例数据后返回生成结果、字段命中情况和格式告警。
- 新增差异预览接口，对比“当前生效模板”和“待保存模板”的输出差异。
- 为模板保存增加版本快照，至少保留最近一次已生效版本，便于回看。

### M8-2 前端命名规则交互

- 在系统设置的命名规则区域补充“规则测试”“差异预览”“导入”“导出”入口。
- 双语命名采用显式开关和显示策略，不隐式拼接中英文标题。
- 规则测试弹窗展示输入样例、生成结果、警告信息和非法字符过滤结果。
- 差异预览展示旧文件名、当前模板结果、待保存模板结果和变化字段。
- 模板保存后更新版本信息展示，明确当前生效版本和最近测试时间。

### M8-3 预览链路联动

目标：让每条命名预览都能说明“由哪个模板版本生成”，模板更新后不自动改写历史预览，但能明确提示用户是否需要重新生成。

后端设计：

- `rename_previews` 增加模板版本快照字段，建议字段：
  - `naming_template_type`：`movie` 或 `episode`，记录生成时使用的模板类型。
  - `naming_template_version`：生成预览时的模板版本号。
  - `naming_template_updated_at`：生成预览时模板配置更新时间，便于排查版本来源。
- 旧库迁移时字段允许为空；旧预览记录没有模板版本时按“未知版本”展示，不强制判定为过期。
- `generate_rename_previews` 生成或覆盖预览时，根据解析后的媒体类型写入对应模板版本快照。
- `list_rename_previews` 返回模板版本快照和当前模板版本对比状态，至少包含：
  - 预览使用的模板版本。
  - 当前生效模板版本。
  - 是否基于旧模板生成。
  - 是否缺少模板版本快照。
- TMDB、IMDb、AI 和人工候选回填仍复用现有命名构建逻辑；候选确认时如重新生成目标名，也要保留或更新对应模板版本快照。

前端设计：

- 重命名预览列表展示轻量状态提示：
  - 当前版本：不额外打扰。
  - 旧模板：显示“旧模板生成”提示，可引导用户重新生成预览。
  - 未知版本：显示“未记录模板版本”，避免误判。
- 提示不阻断编辑、TMDB 匹配、AI 匹配和 dry-run。
- 重新生成预览时沿用现有手动编辑保护：存在 `edited_name` 时继续要求用户确认，不自动覆盖。
- 150% 浏览器缩放下，状态提示不得挤压核心操作列；优先使用短标签和 tooltip 展示完整说明。

验收重点：

- 新生成预览记录能写入模板版本快照。
- 模板保存并升版后，旧预览能展示旧模板提示。
- 未带版本的历史记录可以正常展示和执行原有操作。
- 手动编辑过目标名的记录在重新生成前仍有确认保护。
- M4 TMDB、M6 AI、M7 标题识别和手动排除流程不因新增字段回退。

## 4. 重点设计约束

- 不改变现有设置保存接口的基本调用方式，优先做向后兼容扩展。
- 不把模板测试结果直接写入正式命名预览数据表。
- 不因为模板升级自动批量改写历史预览或历史重命名记录。
- 双语命名只处理模板和预览表达，不在本阶段扩展新的外部元数据站点。
- 历史预览只做状态提示，不自动触发重生成、不自动覆盖用户编辑。
- 预览模板版本快照是生成时事实记录，不作为权限、审计或回滚系统的替代。

## 5. 不做范围

- 不做多用户模板协作。
- 不做模板审批流。
- 不做按媒体源绑定不同模板版本。
- 不做远程模板仓库同步。
- 不做完整回滚体系；正式回滚治理仍放到 M9。

# M10 多站点元数据源验收报告

版本：`0.10.7`
日期：2026-07-12

## 1. 验收结论

M10 多站点元数据源已完成开发验收范围。

已完成：

- 元数据源配置表、配置 API、脱敏保存和连接测试。
- Metadata Provider registry 和多源匹配编排。
- Bangumi 官方 API 补充源。
- TVDB 官方 API 补充源。
- 用户自备豆瓣代理补充源。
- 多源候选去重、字段完整度、来源优先级和字段来源记录。
- 重命名页单条、选中行和当前列表多源匹配入口。
- 系统设置“刮削设置”补充元数据源配置卡片。
- 候选弹窗来源标签和字段来源展示。
- 多源匹配复用敏感词和外部提交保护。
- 覆盖提交需要强确认并记录原因和审计信息。

## 2. 已验收能力

### Provider 配置

- 支持 Bangumi、TVDB、豆瓣代理启用、优先级、Base URL、API Key、超时和重试。
- API Key / Token 不通过列表接口返回明文。
- Bangumi 和 TVDB Base URL 固定为官方地址，避免密钥被转发到第三方主机。
- 豆瓣代理必须由用户自行配置 Base URL，系统不内置第三方代理地址。

### 多源匹配

- 支持 `primary_only`、`fallback`、`parallel` 编排模式。
- 单站点失败不会阻断其他站点。
- 批量多源匹配返回条目级和 Provider 级摘要。
- 当前前端“当前列表全部多源匹配”基于页面已加载列表执行。

### 候选治理

- 候选保留 `merge_key`、`source_priority`、`field_completeness`、`field_sources`。
- 疑似同一作品的多源候选可合并，并保留更完整字段。
- 候选确认回填后继续按当前命名模板生成目标名并刷新模板状态。

### 外部提交保护

- 文件名、路径、解析标题命中敏感词时不会调用外部 Provider。
- 拦截记录进入“敏感词设置”的外部提交拦截列表。
- 用户选择继续提交时必须填写风险确认原因。
- 覆盖提交会写入处理记录和审计事件。

## 3. 未纳入 M10 的范围

# M10 多站点元数据源验收清单

## 0. 验收口径

- M10 只验收多站点元数据源和多源候选治理能力。
- WebDAV、FTP、SFTP、S3 仍属于 M11，不纳入 M10 验收。
- 所有外部提交必须继续遵守敏感词和外部提交保护。
- 截至 `0.10.6`，Bangumi、TVDB 和用户自备豆瓣代理真实搜索、前端配置入口、重命名页多源匹配入口和候选字段来源展示已完成。

## 1. Provider 配置

- [x] TMDB 原有配置和测试连接不回退。
- [x] 可通过后端配置 API 启用、禁用 Bangumi Provider。
- [x] 可通过后端配置 API 启用、禁用 TVDB Provider。
- [x] 可通过后端配置 API 启用、禁用豆瓣代理 Provider。
- [x] 豆瓣代理 Base URL 未配置时不会发起配置级测试请求。
- [x] API Key、Token 等敏感值列表接口不返回明文。
- [x] Provider 配置级测试返回明确成功或失败提示。
- [x] 前端设置页展示并编辑 Bangumi、TVDB、豆瓣代理配置。

## 2. 多源编排

- [x] 后端支持只使用主站匹配。
- [x] 后端支持 TMDB 未匹配或低置信时进入补充源编排。
- [x] 后端支持多源并行搜索模式。
- [x] 单个站点失败不阻断其他站点。
- [x] 所有站点失败时保留本地解析结果。
- [x] 批量多源匹配返回成功、失败、拦截和跳过摘要。
- [x] 前端提供多源匹配入口和批量确认提示。

## 3. Bangumi

- [x] 动漫条目可调用 Bangumi 搜索。
- [x] Bangumi 候选使用统一候选结构。
- [x] Bangumi 候选包含来源标签和匹配原因。
- [x] Bangumi 失败时不会影响 TMDB 结果。
- [x] Bangumi 请求显式排除 NSFW 条目。
- [x] Bangumi Access Token 加密保存且不通过 API 返回明文。
- [x] Bangumi 连接测试调用真实搜索接口并返回中文结果。
- [x] Bangumi Token 只会发送到官方 HTTPS 主机。
- [x] Bangumi 连接测试要求系统设置写权限。
- [x] Bangumi Token 可显式清除并恢复匿名访问。

## 4. TVDB

- [x] 剧集条目可调用 TVDB 搜索。
- [x] TVDB 可补充分季分集字段。
- [x] TVDB 候选展示来源标签和匹配原因。
- [x] TVDB 失败时不会影响 TMDB 结果。
- [x] TVDB API Key 加密保存且不通过 API 返回明文。
- [x] TVDB Base URL 只允许官方 `https://api4.thetvdb.com/v4`。

## 5. 豆瓣代理

- [x] 国产影视或中文标题条目可调用用户配置的豆瓣代理。
- [x] 系统不内置第三方豆瓣代理地址。
- [x] 豆瓣代理候选展示来源标签和匹配原因。
- [x] 豆瓣代理失败时给出明确中文错误。
- [x] 豆瓣代理 API Key 加密保存且不通过 API 返回明文。
- [x] 豆瓣代理未配置 Base URL 时不发起连接测试。

## 6. 候选合并和回填

- [x] 多源候选具备 `merge_key` 并可去重。
- [x] 不同站点疑似同一作品的候选可在后端合并为更完整候选。
- [x] 候选数据包含字段来源 `field_sources`。
- [x] 候选排序综合标题、年份、类型、季集、站点优先级和字段完整度。
- [x] 候选确认回填后按当前命名模板生成目标名。
- [x] 候选确认回填后保留或刷新模板版本快照。
- [x] 前端候选详情展示字段来源。

## 7. 外部提交保护

- [x] 命中文件名敏感词时不提交任何元数据源。
- [x] 命中路径敏感词时不提交任何元数据源。
- [x] 命中解析标题敏感词时不提交任何元数据源。
- [x] 拦截记录包含目标服务和拦截原因。
- [ ] 用户覆盖提交仍需强确认和记录原因。

# M10 多站点元数据源验收清单

## 0. 验收口径

- M10 只验收多站点元数据源和多源候选治理能力。
- WebDAV、FTP、SFTP、S3 仍属于 M11，不纳入 M10 验收。
- 所有外部提交必须继续遵守敏感词和外部提交保护。

## 1. Provider 配置

- [ ] TMDB 原有配置和测试连接不回退。
- [ ] 可启用、禁用 Bangumi Provider。
- [ ] 可启用、禁用 TVDB Provider。
- [ ] 可启用、禁用豆瓣代理 Provider。
- [ ] 豆瓣代理 Base URL 未配置时不会发起请求。
- [ ] API Key、Token 等敏感值列表接口不返回明文。
- [ ] Provider 连接测试返回明确成功或失败提示。

## 2. 多源编排

- [ ] 支持只使用主站匹配。
- [ ] 支持 TMDB 未匹配或低置信时调用补充源。
- [ ] 支持用户手动触发多源并行搜索。
- [ ] 单个站点失败不阻断其他站点。
- [ ] 所有站点失败时保留本地解析结果。
- [ ] 批量多源匹配返回成功、失败、拦截和跳过摘要。

## 3. Bangumi

- [ ] 动漫条目可调用 Bangumi 搜索。
- [ ] Bangumi 候选使用统一候选结构。
- [ ] Bangumi 候选展示来源标签和匹配原因。
- [ ] Bangumi 失败时不会影响 TMDB 结果。

## 4. TVDB

- [ ] 剧集条目可调用 TVDB 搜索。
- [ ] TVDB 可补充分季分集字段。
- [ ] TVDB 候选展示来源标签和匹配原因。
- [ ] TVDB 失败时不会影响 TMDB 结果。

## 5. 豆瓣代理

- [ ] 国产影视或中文标题条目可调用用户配置的豆瓣代理。
- [ ] 系统不内置第三方豆瓣代理地址。
- [ ] 豆瓣代理候选展示来源标签和匹配原因。
- [ ] 豆瓣代理失败时给出明确中文错误。

## 6. 候选合并和回填

- [ ] 同一站点同一 `provider_id` 候选会去重。
- [ ] 不同站点疑似同一作品的候选可合并展示。
- [ ] 候选详情展示字段来源。
- [ ] 候选排序综合标题、年份、类型、季集、站点优先级和字段完整度。
- [ ] 候选确认回填后按当前命名模板生成目标名。
- [ ] 候选确认回填后保留或刷新模板版本快照。

## 7. 外部提交保护

- [ ] 命中文件名敏感词时不提交任何元数据源。
- [ ] 命中路径敏感词时不提交任何元数据源。
- [ ] 命中解析标题敏感词时不提交任何元数据源。
- [ ] 拦截记录包含目标 Provider 和拦截原因。
- [ ] 用户覆盖提交仍需强确认和记录原因。

## 8. 回归

- [ ] M4 TMDB V4/V3 连接和匹配正常。
- [ ] M6 AI 解析和敏感词保护正常。
- [ ] M7 TMDB→AI 后备链路正常。
- [ ] M8 命名模板版本快照正常。

## 9. 自动化验证

- [ ] `npm.cmd run backend:test` 通过。
- [ ] `npm.cmd run frontend:test` 通过。
- [ ] `npm.cmd run frontend:build` 通过。
- [ ] `npm.cmd run check:encoding` 通过。
- [ ] `git diff --check` 通过。

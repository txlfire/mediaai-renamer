# M7 增量扫描开发计划

## 1. 阶段目标

M7 将增量扫描列为下一阶段主线能力。目标是在重复扫描同一媒体源时，跳过未变化文件，只处理新增、修改和需要重新处理的文件。

## 2. 拆分建议

### M7-0 设计和迁移

- 新增 `scan_file_index` 表。
- 扩展 `scan_jobs` 扫描模式和统计字段。
- 补充旧库迁移测试。
- 保持旧 API 默认全量扫描兼容。

### M7-1 后端增量扫描服务

- 生成轻量文件指纹。
- 实现全量扫描写索引。
- 实现增量扫描比对索引。
- 统计新增、修改、跳过、缺失数量。
- 扫描任务结果返回增量摘要。

### M7-2 重命名联动

- 真实重命名成功后更新扫描索引。
- 原路径标记为 renamed。
- 目标路径写入 active。
- 失败时记录警告，不回滚已完成真实重命名。
- 重命名预览解析增强：文件名标题不足时，从所属文件夹名补充电影名或剧集名，并记录解析来源。
- 重命名预览列表增加手动排除能力，将用户排除的文件转入待处理文件列表。

### M7-3 前端交互

- 媒体源选择后查询扫描模式建议。
- 扫描任务页增加全量 / 增量选择。
- 扫描任务列表展示增量统计。
- 扫描详情展示跳过、缺失和新增修改数量。
- 重命名页增加 AI 批量匹配入口。
- 重命名页增加 TMDB 后备 AI 串联匹配模式，用户可选择“仅 TMDB”“仅 AI”“TMDB 未匹配时自动 AI 二次匹配”。
- 批量 AI 和串联匹配执行前显示详尽确认提示，包含处理数量、提交字段范围、目标外部服务、敏感词保护、Token 成本、失败回退和不会自动真实重命名说明。
- 重命名预览详情展示标题来源：文件名、所属文件夹或混合解析。
- 重命名预览操作区新增“排除”入口，支持单条和批量排除到待处理文件列表。
- 待处理文件列表新增“手动排除”原因展示。

### M7-3.5 匹配链路治理

- 后端新增批量 AI 解析服务，复用 M6 外部提交保护。
- 后端新增 TMDB 后备 AI 编排逻辑，只对 TMDB 未匹配或低置信条目调用 AI。
- 批量匹配返回成功、失败、拦截、跳过统计和逐条日志。
- 批量结果保留单条回填入口，不默认覆盖用户已编辑目标文件名。
- 增加 Token 用量汇总，便于用户评估 AI 成本。
- 最小扫描文件阈值回归验证：低于阈值的视频只进入待处理列表，不进入扫描结果和命名预览。

### M7-4 文档和验收

- 更新 M7 用户手册。
- 更新完整用户手册。
- 更新 README 文档索引。
- 执行后端测试、前端测试、前端构建、编码检查和打包验证。

## 3. 不做范围

- 不做文件内容哈希。
- 不做目录实时监听。
- 不做 WebSocket 进度推送。
- 不做跨媒体源去重。
- 不自动清理历史索引。

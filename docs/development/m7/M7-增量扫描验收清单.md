# M7 增量扫描验收清单

## 1. 数据迁移

- [x] 新库会创建 `scan_file_index` 表。
- [x] 旧库升级后会创建索引表和新增扫描任务统计字段。
- [x] 旧前端不传 `scan_mode` 时仍能创建全量扫描任务。

## 2. 全量扫描

- [x] 首次扫描默认使用全量扫描。
- [x] 全量扫描写入扫描结果。
- [x] 全量扫描建立或刷新扫描索引。
- [x] 全量扫描会将本次未看到的历史文件标记为缺失。

## 3. 增量扫描

- [x] 存在历史索引时，后端可建议使用增量扫描。
- [x] 未变化文件被跳过并计入 `skipped_count`。
- [x] 新增文件进入本次扫描结果并计入 `new_count`。
- [x] 文件大小变化后进入本次扫描结果并计入 `changed_count`。
- [x] 文件修改时间变化后进入本次扫描结果并计入 `changed_count`。
- [x] 历史存在但本次未发现的文件计入 `missing_count`。

## 4. 重命名联动

- [x] 真实重命名成功后，原路径索引标记为 `renamed`。
- [x] 真实重命名成功后，目标路径索引标记为 `active`。
- [x] 后续增量扫描不会再把已重命名的原路径误判为缺失待处理。
- [x] 重命名后目标文件仍可被后续增量扫描识别。
- [x] 文件名只有季集号时，命名预览可从所属文件夹名补充剧集名。
- [x] 文件名只有技术参数或标题为空时，命名预览可从所属文件夹名补充电影名。
- [x] 文件名和文件夹名标题冲突时，不自动覆盖文件名标题，并标记为需检查。
- [x] 标题识别来源模式支持 `file_name_first`、`parent_folder_fallback`、`parent_folder_first`、`manual_only`。
- [x] 默认模式为 `parent_folder_fallback`，剧集文件名标题不足时优先用上级文件夹兜底。
- [x] 全量扫描和增量扫描使用同一套标题识别模式。
- [x] 增量扫描跳过未变化文件时，不重复生成识别结果。
- [x] 手动重新生成预览时，会按当前标题识别模式重新解析。
- [x] TMDB / IMDb / AI 匹配前可区分原文件名、解析标题和上级文件夹标题。
- [x] 上级文件夹标题命中敏感词时，外部匹配提交同样被拦截并写入记录。
- [x] 命名预览详情能展示标题解析来源。
- [x] 命名预览详情能展示当前识别模式、上级文件夹候选标题和标题冲突原因。
- [x] 单条预览可手动排除到待处理文件列表。
- [x] 批量选中的预览可手动排除到待处理文件列表。
- [x] 排除操作只调整任务记录，不删除真实文件。
- [x] 手动排除的文件在待处理列表中显示“手动排除”原因。

## 5. 前端交互

- [x] 媒体源已有索引时，扫描任务页面展示全量 / 增量选择。
- [x] 首次扫描时页面提示需要全量扫描。
- [x] 增量扫描完成后，任务列表展示新增、修改、跳过、缺失统计。
- [x] 增量扫描失败时展示中文错误，不影响历史索引。
- [x] 重命名页支持单条、选中批量、全部未匹配的 AI 匹配入口。
- [x] 重命名页支持“仅 TMDB”“仅 AI”“TMDB 未匹配时自动 AI 二次匹配”三种匹配模式。
- [x] 批量 AI 和 TMDB 后备 AI 执行前展示详尽确认提示。
- [x] 确认提示包含处理数量、提交字段范围、外部服务、敏感词保护、Token 成本、失败处理和不会自动真实重命名说明。
- [x] 批量匹配完成后展示成功、失败、拦截、跳过摘要。
- [x] 批量 AI 结果保留逐条回填入口，不自动覆盖用户已编辑目标文件名。

## 6. 匹配链路保护

- [x] 后端提供统一 AI Provider 抽象，单条 AI、批量 AI、连接测试共用同一调用入口。
- [x] 设置页支持新增、编辑、删除、测试 AI Provider。
- [x] 设置页支持一键切换当前激活模型，切换后新 AI 请求使用新模型。
- [x] Provider 列表接口不返回明文 API Key。
- [x] 批量 AI 确认弹窗展示当前 Provider、模型名、Base URL 脱敏信息和预计处理数量。
- [x] AI 候选结果保存 `provider_id`、`provider_name`、`model_name`。
- [x] 切换模型不影响历史 AI 候选和已回填结果。
- [x] 当前激活 Provider 未配置或连接测试失败时，批量 AI 入口不可直接执行。
- [x] 命中敏感词的条目不会提交到 TMDB。
- [x] 命中敏感词的条目不会提交到 AI。
- [x] TMDB 后备 AI 模式只对 TMDB 未匹配或低置信条目发起 AI 请求。
- [x] 单条失败不阻断批量中其他条目。
- [x] 批量 AI 匹配返回 Token 用量汇总。
- [x] 低于 `scan.minimum_file_size` 的视频不会进入正常扫描结果。
- [x] 低于 `scan.minimum_file_size` 的视频会进入待处理文件列表，原因为 `size_filtered`。
- [x] 旧扫描任务中的历史小文件不会被误判为新扫描阈值失效。

# M7 增量扫描验收清单

## 1. 数据迁移

- [ ] 新库会创建 `scan_file_index` 表。
- [ ] 旧库升级后会创建索引表和新增扫描任务统计字段。
- [ ] 旧前端不传 `scan_mode` 时仍能创建全量扫描任务。

## 2. 全量扫描

- [ ] 首次扫描默认使用全量扫描。
- [ ] 全量扫描写入扫描结果。
- [ ] 全量扫描建立或刷新扫描索引。
- [ ] 全量扫描会将本次未看到的历史文件标记为缺失。

## 3. 增量扫描

- [ ] 存在历史索引时，后端可建议使用增量扫描。
- [ ] 未变化文件被跳过并计入 `skipped_count`。
- [ ] 新增文件进入本次扫描结果并计入 `new_count`。
- [ ] 文件大小变化后进入本次扫描结果并计入 `changed_count`。
- [ ] 文件修改时间变化后进入本次扫描结果并计入 `changed_count`。
- [ ] 历史存在但本次未发现的文件计入 `missing_count`。

## 4. 重命名联动

- [ ] 真实重命名成功后，原路径索引标记为 `renamed`。
- [ ] 真实重命名成功后，目标路径索引标记为 `active`。
- [ ] 后续增量扫描不会再把已重命名的原路径误判为缺失待处理。
- [ ] 重命名后目标文件仍可被后续增量扫描识别。
- [ ] 文件名只有季集号时，命名预览可从所属文件夹名补充剧集名。
- [ ] 文件名只有技术参数或标题为空时，命名预览可从所属文件夹名补充电影名。
- [ ] 文件名和文件夹名标题冲突时，不自动覆盖文件名标题，并标记为需检查。
- [ ] 命名预览详情能展示标题解析来源。
- [ ] 单条预览可手动排除到待处理文件列表。
- [ ] 批量选中的预览可手动排除到待处理文件列表。
- [ ] 排除操作只调整任务记录，不删除真实文件。
- [ ] 手动排除的文件在待处理列表中显示“手动排除”原因。

## 5. 前端交互

- [ ] 媒体源已有索引时，扫描任务页面展示全量 / 增量选择。
- [ ] 首次扫描时页面提示需要全量扫描。
- [ ] 增量扫描完成后，任务列表展示新增、修改、跳过、缺失统计。
- [ ] 增量扫描失败时展示中文错误，不影响历史索引。
- [ ] 重命名页支持单条、选中批量、全部未匹配的 AI 匹配入口。
- [ ] 重命名页支持“仅 TMDB”“仅 AI”“TMDB 未匹配时自动 AI 二次匹配”三种匹配模式。
- [ ] 批量 AI 和 TMDB 后备 AI 执行前展示详尽确认提示。
- [ ] 确认提示包含处理数量、提交字段范围、外部服务、敏感词保护、Token 成本、失败处理和不会自动真实重命名说明。
- [ ] 批量匹配完成后展示成功、失败、拦截、跳过摘要。
- [ ] 批量 AI 结果保留逐条回填入口，不自动覆盖用户已编辑目标文件名。

## 6. 匹配链路保护

- [ ] 命中敏感词的条目不会提交到 TMDB。
- [ ] 命中敏感词的条目不会提交到 AI。
- [ ] TMDB 后备 AI 模式只对 TMDB 未匹配或低置信条目发起 AI 请求。
- [ ] 单条失败不阻断批量中其他条目。
- [ ] 批量 AI 匹配返回 Token 用量汇总。
- [ ] 低于 `scan.minimum_file_size` 的视频不会进入正常扫描结果。
- [ ] 低于 `scan.minimum_file_size` 的视频会进入待处理文件列表，原因为 `size_filtered`。
- [ ] 旧扫描任务中的历史小文件不会被误判为新扫描阈值失效。

## 7. 回归

- [ ] M1 媒体源、扫描任务、扫描结果正常。
- [ ] M2 命名预览正常。
- [ ] M3 dry-run 和安全重命名正常。
- [ ] M4 TMDB 匹配正常。
- [ ] M5 SMB / NFS 校验正常。
- [ ] M6 AI 解析和敏感词保护正常。
